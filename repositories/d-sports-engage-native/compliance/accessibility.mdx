# Accessibility Compliance

D-Sports Engage supports all 9 Apple Accessibility Nutrition Label categories for iPhone and iPad. This document details the implementation status and technical approach for each feature.

## Feature Support Matrix

| Feature | Status | Details |
|---------|--------|---------|
| **VoiceOver** | ✅ Supported | All interactive elements have `accessibilityRole`, `accessibilityLabel`, `accessibilityHint`, and `accessibilityState` props. |
| **Voice Control** | ✅ Supported | All buttons and controls are labeled for voice command interaction. |
| **Larger Text** | ✅ Supported | Respects iOS Dynamic Type via `AccessibilityInfo`. In-app toggle provides 1.25× scaling. Capped at 2× via `maxFontSizeMultiplier`. |
| **Dark Interface** | ✅ Supported | Full dark/light theme support via `ThemeProvider`. Default is dark mode. |
| **Differentiate Without Color** | ✅ Supported | Directional arrows (▲/▼) alongside price change colors. Text labels on rarity badges. Icons with status indicators. |
| **Sufficient Contrast** | ✅ Supported | High-contrast color variants in `theme/colors.ts`. Activated by in-app toggle or iOS Bold Text setting. Meets WCAG AA (4.5:1). |
| **Reduce Motion** | ✅ Supported | `ReducedMotionConfig` from `react-native-reanimated` with `ReduceMotion.System` mode. Also respects in-app toggle. |
| **Captions** | 🔵 Planned | Will be added when video content is introduced. |
| **Audio Descriptions** | 🔵 Planned | Will be added alongside captions. |

## Web accessibility

In addition to native accessibility features, the web build includes browser-level a11y support:

| Feature | Implementation |
|---------|---------------|
| **Skip to content** | Hidden `<a href="#main-content">` link in `app/+html.tsx`, visible on keyboard focus with styled focus ring. |
| **Keyboard focus outlines** | Global `*:focus-visible` CSS rule applies a `2px solid #4169E1` outline to all focusable elements. |
| **Reduced motion (CSS)** | `@media (prefers-reduced-motion: reduce)` disables CSS animations and transitions site-wide. |
| **High contrast (CSS)** | `@media (prefers-contrast: more)` boosts `--text-secondary-alpha` and `--border-alpha` CSS variables. |
| **Browser media query sync** | `AccessibilityProvider` listens to `prefers-reduced-motion` and `prefers-contrast: more` change events so in-app React state stays in sync with browser settings. |
| **Hover feedback** | `WebHoverWrapper` provides opacity/scale hover states for interactive elements on web; no-ops on native. |

## Architecture

### AccessibilityProvider

The central context (`context/accessibility-context.tsx`) manages all accessibility state across native and web:

```typescript
interface AccessibilityContextType {
  // In-app toggles
  reduceMotion: boolean;
  largerText: boolean;
  highContrast: boolean;
  screenReaderSupport: boolean;
  hapticFeedback: boolean;

  // Derived values
  textScale: number;          // max(inAppScale, systemFontScale)
  systemFontScale: number;    // iOS Dynamic Type multiplier
  isScreenReaderActive: boolean;
  isSystemReduceMotion: boolean;
  isSystemBoldText: boolean;
}
```

**Key behaviors:**
- `textScale` = `Math.max(inAppToggle ? 1.25 : 1.0, systemFontScale)`
- `reduceMotion` = `inAppToggle || isSystemReduceMotion` (native `AccessibilityInfo` or web `prefers-reduced-motion`)
- `highContrast` = `inAppToggle || isSystemBoldText` (native bold text or web `prefers-contrast: more`)

### AccessibleText Component

`components/ui/AccessibleText.tsx` wraps `<Text>` with:
- Font size scaling via `textScale` from context
- Font weight bumping when `highContrast` is active
- `maxFontSizeMultiplier={2}` to prevent layout breakage
- `allowFontScaling={true}` for native Dynamic Type

### ReducedMotionConfig

Added to `app/_layout.tsx` at the root level:

```tsx
<ReducedMotionConfig mode={ReduceMotion.System}>
  <AccessibilityProvider>
    {/* ... app tree */}
  </AccessibilityProvider>
</ReducedMotionConfig>
```

This globally tells `react-native-reanimated` to skip animations when the system "Reduce Motion" setting is enabled. On web, the CSS `@media (prefers-reduced-motion: reduce)` rule in `app/+html.tsx` provides an additional layer that disables CSS animations and transitions.

### High Contrast Colors

`theme/colors.ts` exports `darkHighContrast` and `lightHighContrast` variants with boosted alpha values:

| Token | Default (Dark) | High Contrast (Dark) |
|-------|---------------|---------------------|
| `textSecondary` | `rgba(255,255,255,0.7)` | `rgba(255,255,255,0.9)` |
| `textMuted` | `rgba(255,255,255,0.5)` | `rgba(255,255,255,0.75)` |
| `borderLight` | `rgba(255,255,255,0.1)` | `rgba(255,255,255,0.25)` |

Use `getColors(isDark, highContrast)` to get theme-aware colors with optional contrast boost.

## App Store Connect

After submitting the next app update:

1. Go to **App Store Connect → App → General → App Information**
2. Under **Accessibility**, check all supported features for iPhone and iPad
3. Set **Accessibility URL** to `https://d-sports.org/accessibility`
4. Submit with the update
