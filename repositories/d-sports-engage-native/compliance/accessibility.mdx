# Accessibility Compliance

D-Sports Engage supports all 9 Apple Accessibility Nutrition Label categories for iPhone and iPad. This document details the implementation status and technical approach for each feature.

## Feature Support Matrix

| Feature | Status | Details |
|---------|--------|---------|
| **VoiceOver** | ✅ Supported | All interactive elements have `accessibilityRole`, `accessibilityLabel`, `accessibilityHint`, and `accessibilityState` props. |
| **Voice Control** | ✅ Supported | All buttons and controls are labeled for voice command interaction. |
| **Larger Text** | ✅ Supported | Respects iOS Dynamic Type via `AccessibilityInfo`. In-app toggle provides 1.25× scaling. Capped at 2× via `maxFontSizeMultiplier`. |
| **Dark Interface** | ✅ Supported | Full dark/light theme support via `ThemeProvider`. Default is dark mode. |
| **Differentiate Without Color** | ✅ Supported | Directional arrows (▲/▼) alongside price change colors. Text labels on rarity badges. Icons with status indicators. Color blindness mode remaps semantic colors for protanopia, deuteranopia, and tritanopia via `useColors()`. |
| **Sufficient Contrast** | ✅ Supported | High-contrast color variants in `theme/colors.ts`. Activated by in-app toggle or iOS Bold Text setting. Meets WCAG AA (4.5:1). High contrast mode also activates the color-blind safe palette. |
| **Reduce Motion** | ✅ Supported | `ReducedMotionConfig` from `react-native-reanimated` with `ReduceMotion.System` mode. Also respects in-app toggle. |
| **Captions** | 🔵 Planned | Will be added when video content is introduced. |
| **Audio Descriptions** | 🔵 Planned | Will be added alongside captions. |

## Architecture

### AccessibilityProvider

The central context (`context/accessibility-context.tsx`) manages all accessibility state:

```typescript
interface AccessibilityContextType {
  // In-app toggles
  reduceMotion: boolean;
  largerText: boolean;
  highContrast: boolean;
  screenReaderSupport: boolean;
  hapticFeedback: boolean;
  colorBlindModeEnabled: boolean;
  colorBlindModeType: "protanopia" | "deuteranopia" | "tritanopia";

  // Derived values
  textScale: number;          // max(inAppScale, systemFontScale)
  systemFontScale: number;    // iOS Dynamic Type multiplier
  isScreenReaderActive: boolean;
  isSystemReduceMotion: boolean;
  isSystemBoldText: boolean;
}
```

**Key behaviors:**
- `textScale` = `Math.max(inAppToggle ? 1.25 : 1.0, systemFontScale)`
- `reduceMotion` = `inAppToggle || isSystemReduceMotion`
- `highContrast` = `inAppToggle || isSystemBoldText`

### AccessibleText Component

`components/ui/AccessibleText.tsx` wraps `<Text>` with:
- Font size scaling via `textScale` from context
- Font weight bumping when `highContrast` is active
- `maxFontSizeMultiplier={2}` to prevent layout breakage
- `allowFontScaling={true}` for native Dynamic Type

### ReducedMotionConfig

Added to `app/_layout.tsx` at the root level:

```tsx
<ReducedMotionConfig mode={ReduceMotion.System}>
  <AccessibilityProvider>
    {/* ... app tree */}
  </AccessibilityProvider>
</ReducedMotionConfig>
```

This globally tells `react-native-reanimated` to skip animations when the iOS "Reduce Motion" setting is enabled.

### High Contrast Colors

`theme/colors.ts` exports `darkHighContrast` and `lightHighContrast` variants with boosted alpha values:

| Token | Default (Dark) | High Contrast (Dark) |
|-------|---------------|---------------------|
| `textSecondary` | `rgba(255,255,255,0.7)` | `rgba(255,255,255,0.9)` |
| `textMuted` | `rgba(255,255,255,0.5)` | `rgba(255,255,255,0.75)` |
| `borderLight` | `rgba(255,255,255,0.1)` | `rgba(255,255,255,0.25)` |

Use `getColors(isDark, { highContrast, colorBlindnessEnabled, colorBlindnessType })` to get theme-aware colors with optional contrast boost and color blindness adaptation.

### Color blindness mode

The app supports three color vision deficiency (CVD) profiles that remap semantic colors (positive/negative trends, danger states, and reputation indicators) to distinguishable pairs:

| Profile | Positive | Negative | Use case |
|---------|----------|----------|----------|
| **Deuteranopia** (default) | Blue `#2563EB` | Orange `#F97316` | Green-blind (most common) |
| **Protanopia** | Deep blue `#1D4ED8` | Deep orange `#EA580C` | Red-blind |
| **Tritanopia** | Sky blue `#0EA5E9` | Purple `#D946EF` | Blue-yellow confusion |

Color blindness mode activates when either `colorBlindModeEnabled` is toggled on or `highContrast` is enabled. The user selects their CVD profile under **Settings → Accessibility → Color blindness type**.

### `useColors` hook

`hooks/use-colors.ts` is the recommended way to access theme-aware, accessibility-adapted colors throughout the app:

```typescript
import { useColors } from "@/hooks/use-colors";

function MyComponent() {
  const colors = useColors();
  // colors.positive  -> green normally, adapted per selected CVD profile
  // colors.negative  -> red normally, adapted per selected CVD profile
}
```

The hook reads the current color scheme, high-contrast state, and color blindness preferences from context and returns the resolved palette via `getColors()`. All semantic color tokens (`positive`, `negative`, `danger`, `heel`) automatically adapt when color blindness mode is active.

## App Store Connect

After submitting the next app update:

1. Go to **App Store Connect → App → General → App Information**
2. Under **Accessibility**, check all supported features for iPhone and iPad
3. Set **Accessibility URL** to `https://d-sports.org/accessibility`
4. Submit with the update
