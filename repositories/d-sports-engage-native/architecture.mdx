## Routing

The app uses **Expo Router 6** with file-based routing and typed routes enabled.

- **Route groups:** `(tabs)` for main navigation, `(auth)` for login/signup, `(onboarding)` for new users.
- **Stack routes:** Leaderboard uses `/leaderboard/[teamSlug]` and locker room uses `/locker-room/wheel-spin` and `/locker-room/player-guess`.
- **Auth protection:** `_layout.tsx` automatically redirects based on authentication and onboarding state.
- **Path alias:** `@/*` maps to the project root.

## State and context boundaries

| Layer | Technology | Scope |
| ----- | ---------- | ----- |
| Global store | Zustand + MMKV | Theme, cart, points |
| Auth/profile | React Context (`UserContext`) | Session, user data, team membership |
| Collectibles | React Context (`CollectiblesContext`) | Owned packs and items |
| UI visibility | React Context (`NavBarVisibilityContext`) | Navbar hide/show on scroll |
| Accessibility | React Context (`AccessibilityContext`) | Accessibility preferences |
| Ephemeral state | Feature hooks | Wallet, shop, feed screen state |

## Modular screen pattern

Screen files contain **only JSX**. All state, effects, and handlers live in dedicated hooks:

- `use-wallet-screen.ts` — wallet/token fetch, PIN verification, transaction handlers
- `use-shop-screen.ts` — cart state, product queries, carousel auto-scroll, checkout
- `use-feed-section.ts` — locker room feed loading, interactions, modals

Sub-components are extracted into `components/<feature>/` directories with barrel exports.

## API client and retry behavior

- All network calls route through `lib/api/*` modules.
- `lib/api/client.ts` handles auth token propagation, error normalization, and retry strategy.
- `lib/api/cache.ts` provides MMKV cache-first fetching with configurable TTL.
- Quest/reward/team flows consume backend eligibility as source of truth.

## Checkout and payments

| Channel | Technology | Platforms |
| ------- | ---------- | --------- |
| Fiat (IAP) | RevenueCat | iOS (Apple IAP), Android (Google Play), Web (Stripe) |
| Crypto | Thirdweb SDK | All (calls PWA backend for on-chain payments) |

- Crypto checkout calls `POST /api/checkout/crypto` and `POST /api/checkout/crypto/verify`.
- Supported chains: Arbitrum (default), Ethereum, Polygon.

## Platform-specific behavior

- iOS/Android differences are handled in native-specific components and runtime checks.
- Web target uses `maxWidth: 480px` responsive container (`AppScreen`), web hover states, and PWA manifest (`display: standalone`).
- Haptics, modal controls, and animation use platform-safe fallbacks via Reanimated 4.
- Android uses `softwareKeyboardLayoutMode: resize` for proper keyboard handling.
