## Routing

The app uses **Expo Router 6** with file-based routing and typed routes enabled. Route groups organize the navigation hierarchy:

| Route group       | Purpose                                   |
| ----------------- | ----------------------------------------- |
| `(auth)/`         | Login, signup, SSO callback, password reset |
| `(onboarding)/`   | New user onboarding flow                  |
| `(tabs)/`         | Main 5-tab navigation                     |
| `settings/`       | Settings pages with nested modals/tabs    |

The root layout (`app/_layout.tsx`) wraps the app in provider hierarchy (Clerk, RevenueCat, Thirdweb, user context, theme, accessibility) and handles auth-based redirects.

## Screen-hook pattern

Screen files contain **only imports and JSX**. All state, effects, and handlers live in dedicated hooks:

- `hooks/use-wallet-screen.ts` ‚Äî wallet state, token fetch, PIN verification, transaction handlers
- `hooks/use-shop-screen.ts` ‚Äî cart state, product queries, carousel auto-scroll, checkout logic
- `hooks/use-feed-section.ts` ‚Äî locker room feed loading, pagination, interactions

Sub-components are extracted into `components/wallet/`, `components/shop/`, and `components/locker-room/` with barrel exports via `index.ts`.

## Tab bar architecture

The tab bar implementation differs by platform:

- **iOS** ‚Äî `app/(tabs)/_layout.ios.tsx` uses `NativeTabs` from `expo-router/unstable-native-tabs` for system tab bar behavior. Uses `systemChromeMaterialDark` material with gold selection tint (`#FFD700`).
- **Android and web** ‚Äî `app/(tabs)/_layout.tsx` renders a custom `PillTabBar` overlay with `expo-blur` and tints from `tab-config.ts`.
- **Shared config** ‚Äî `app/(tabs)/tab-config.ts` defines tab order, SF Symbol pairs (iOS), and active-tab pathname matching rules.

## State management

| Layer            | Technology       | Scope                          |
| ---------------- | ---------------- | ------------------------------ |
| Global store     | Zustand 5        | Theme, cart, points            |
| Persistence      | MMKV 4           | Synchronous read/write         |
| Auth and profile | React Context    | `UserContext` in `context/`    |
| Collectibles     | React Context    | `CollectiblesContext`          |
| UI visibility    | React Context    | Navbar visibility, FAB state   |
| Accessibility    | React Context    | Font scale, reduced motion     |

Zustand persistence uses an MMKV adapter (`services/storage.ts`). The store (`services/store.ts`) provides `getItem<T>()`, `setItem()`, and `removeItem()`.

## API client layer

All network calls route through `lib/api/*` modules:

- `lib/api/client.ts` ‚Äî base HTTP client with auth token injection and error normalization
- `lib/api/cache.ts` ‚Äî `fetchWithCache()` MMKV cache-first fetching for offline resilience
- Domain modules: `wallet-api`, `shop-api`, `user-api`, `quests-api`, `leaderboard-api`, `locker-room-api`, `teams-api`, `collectibles-api`, `checkout-api`, `games-api`
- `lib/api/index.ts` ‚Äî `useApi()` hook that exposes all modules

Quest, reward, and team flows consume backend eligibility as the source of truth.

## Theme system

- `theme/colors.ts` ‚Äî brand colors (Gold `#F5C842`, Royal Blue `#4169E1`), dark/light palettes with base dark `#0a0e1a`
- `theme/typography.ts` and `theme/fonts.ts` ‚Äî type scale and font definitions
- `theme/spacing.ts` ‚Äî spacing tokens
- `components/theme-provider.tsx` ‚Äî `useTheme()` hook provides `resolvedTheme`

## Platform-specific behavior

- **iOS** ‚Äî Native tab bar (Liquid Glass), Apple Sign-In, haptic feedback, system push via OneSignal
- **Android** ‚Äî Custom pill tab bar with blur, edge-to-edge layout, adaptive icon
- **Web** ‚Äî PWA-ready (`display: standalone`), responsive `maxWidth: 480px` container (`components/layout/AppScreen.tsx`), web hover states via `WebHoverWrapper`, keyboard navigation support
- Haptics, modal controls, and animations use platform-safe fallbacks via Reanimated 4
