## Overview

**d-sports-engage-native** (package name: `engage-native`) is the native mobile app for D-Sports. It mirrors the core PWA experience on iOS and Android: wallet, shop, leaderboard, locker room, and profile.

- **Run:** `bunx expo start` — then press `a` for Android or `i` for iOS, or scan the QR code with Expo Go.
- **Version:** 1.14.2

## Tech stack

| Category   | Technology                |
| ---------- | ------------------------- |
| Framework  | Expo 54, React Native 0.81, React 19 |
| Auth       | Clerk (Expo)              |
| Payments   | RevenueCat (react-native-purchases) |
| Web3       | Thirdweb                  |
| State      | Zustand + MMKV            |
| UI         | Lucide React Native       |
| Navigation | Expo Router (file-based, typed routes) |
| Push       | OneSignal                 |
| Monitoring | Sentry                    |
| Package    | Bun                       |

## Features

- **Wallet** — tokens, holdings, pack opening, crypto checkout (via PWA backend)
- **Shop** — collectibles, cart, coin bundles, checkout (fiat via RevenueCat, crypto via Thirdweb)
- **Leaderboard** — rankings and filters
- **Locker room** — social feed, quests, daily games (Pick'Em, Spin Wheel, Guess the Player)
- **Profile** — user profile, achievements, followers/following
- **Settings** — appearance, notifications, accessibility, privacy, support
- **Theme** — dark/light mode (default dark)

## Environment variables

All runtime variables use the `EXPO_PUBLIC_` prefix. Create a `.env` file in the repo root:

| Variable | Description |
| -------- | ----------- |
| `EXPO_PUBLIC_CLERK_PUBLISHABLE_KEY` | Clerk authentication publishable key |
| `EXPO_PUBLIC_API_URL` | Backend API base URL (e.g. `https://api.d-sports.org`) |
| `EXPO_PUBLIC_TW_CLIENT_ID` | Thirdweb client ID for Web3 features |
| `EXPO_PUBLIC_REVENUECAT_API_KEY` | RevenueCat API key for in-app purchases |
| `EXPO_PUBLIC_REVENUECAT_APPSTORE_ID` | RevenueCat App Store app ID |
| `EXPO_PUBLIC_REVENUECAT_ENTITLEMENT` | RevenueCat entitlement identifier (e.g. `premium`) |
| `EXPO_PUBLIC_SUPABASE_URL` | Supabase project URL (client-side features only) |
| `EXPO_PUBLIC_SUPABASE_KEY` | Supabase publishable key |

## Architecture

### Screen pattern

Screen files contain **only imports and JSX**. All state, effects, and handlers live in dedicated hooks:

- `hooks/use-wallet-screen.ts` — wallet/token fetching, PIN verification, transaction handlers
- `hooks/use-shop-screen.ts` — cart state, product queries, carousel auto-scroll, checkout logic

Sub-components are extracted into `components/wallet/` and `components/shop/` with barrel exports via `index.ts`.

### Routing

The app uses **Expo Router** with file-based routing and typed routes. Route groups:

- `(tabs)` — main tab navigation
- `(auth)` — login, signup, SSO callback, password reset
- `(onboarding)` — new user onboarding

Auth protection in `_layout.tsx` automatically redirects based on authentication and onboarding state.

### API client layer

All API calls go through `lib/api/`, which provides a `useApi()` hook with domain-specific modules:

```typescript
import { useApi } from "@/lib/api";

const api = useApi();
const result = await api.user.getProfile();
```

Available modules: `user`, `wallet`, `shop`, `checkout`, `quests`, `leaderboard`, `lockerRoom`, `teams`, `collectibles`, `games`.

The client injects auth tokens automatically and uses MMKV-based caching (`lib/api/cache.ts`) for offline fallback.

### Payments and checkout

- **Fiat payments** — RevenueCat handles Apple IAP (native), Google Play (native), and Stripe (web)
- **Crypto checkout** — Thirdweb SDK signs on-chain transactions; the PWA backend processes `POST /api/checkout/crypto` and `POST /api/checkout/crypto/verify`
- Supported chains: Arbitrum (default), Ethereum, Polygon

### State management

- **Zustand** store (`services/store.ts`) with MMKV persistence for theme, cart, and points
- **React Context** providers for auth (`UserContext`), collectibles, navbar visibility, and FAB state

## EAS build profiles

Builds use [EAS Build](https://docs.expo.dev/build/introduction/) with the following profiles defined in `eas.json`:

| Profile | Purpose | Distribution |
| ------- | ------- | ------------ |
| `development` | Dev client with simulator support | Internal |
| `development-device` | Dev client for physical devices | Internal |
| `preview` | Internal testing (APK on Android) | Internal |
| `staging` | Pre-production with auto-increment | Staging channel |
| `production` | App Store / Play Store release | Production channel |

Common build commands:

```bash
bun run build:dev:ios       # Development build for iOS
bun run build:preview:android  # Preview APK for Android
bun run build:prod          # Production build for all platforms
bun run update              # OTA update via EAS Update
bun run submit:ios          # Submit to App Store
```
