## Overview

**d-sports-engage-native** (package name: `engage-native`, version 1.10.10) is the native mobile app for D-Sports. It mirrors the core PWA experience on iOS and Android: wallet, shop, leaderboard, locker room, and profile. The app also runs as a PWA on web with a responsive desktop layout.

- **Run:** `bunx expo start` — then press `a` for Android, `i` for iOS, or scan the QR code with Expo Go.

## Features

- **Wallet** — Token balances, top holdings (real-time value), pack opening, crypto checkout via PWA backend
- **Shop** — Collectibles, 3D model viewer, cart, coin bundles, crypto and fiat checkout
- **Leaderboard** — Rankings and filters
- **Locker room** — Social feed, daily pick'em, spin wheel, guess-the-player games, quests
- **Profile** — User profile, settings, and team membership
- **Onboarding** — New-user onboarding flow with team selection
- **Theme** — Dark/light mode (default dark)
- **Image caching** — Cross-platform image caching for shop and wallet screens

## Architecture

- **Screen-hook separation** — screen files contain only JSX; all state and logic live in dedicated hooks (e.g. `use-wallet-screen.ts`, `use-shop-screen.ts`)
- **File-based routing** via Expo Router with route groups `(tabs)`, `(auth)`, `(onboarding)`
- **Zustand + MMKV** for global state with synchronous persistence
- **React Context** for auth (`UserContext`), collectibles, and navbar visibility
- **API caching** — `fetchWithCache()` serves stale MMKV data immediately, then refreshes from the API with configurable TTL per resource
- **Crypto checkout** via Thirdweb SDK calling the PWA backend for on-chain payments (Arbitrum, Ethereum, Polygon)
- **PWA-ready** — `display: standalone`, responsive `maxWidth: 480px` container on desktop, web hover states
- **Path alias** — `@/*` maps to the project root

## Environment variables

All runtime variables use the `EXPO_PUBLIC_` prefix. Create a `.env` file at the repo root:

| Variable | Purpose |
| -------- | ------- |
| `EXPO_PUBLIC_CLERK_PUBLISHABLE_KEY` | Clerk authentication |
| `EXPO_PUBLIC_API_URL` | d-sports-api backend URL |
| `EXPO_PUBLIC_TW_CLIENT_ID` | Thirdweb client ID (Web3) |
| `EXPO_PUBLIC_REVENUECAT_API_KEY` | RevenueCat API key |
| `EXPO_PUBLIC_REVENUECAT_APPSTORE_ID` | RevenueCat App Store ID |
| `EXPO_PUBLIC_REVENUECAT_ENTITLEMENT` | RevenueCat entitlement name |
| `EXPO_PUBLIC_SUPABASE_URL` | Supabase project URL |
| `EXPO_PUBLIC_SUPABASE_KEY` | Supabase publishable key |

## Getting started

1. Clone the repository and run `bun install`.
2. Copy `.env.example` to `.env` and fill in the values above.
3. Run `bunx expo start` — press `a` for Android, `i` for iOS, or open in a browser for web.
4. Run `bun tsc --noEmit` to check TypeScript.

## EAS builds

The app uses [EAS Build](https://docs.expo.dev/build/introduction/) with these profiles:

| Profile | Command | Use case |
| ------- | ------- | -------- |
| `development` | `bun run build:dev` | Dev client with simulator support |
| `preview` | `bun run build:preview` | Internal distribution (Android APK) |
| `production` | `bun run build:prod` | App Store / Play Store release |

OTA updates are published automatically via GitHub Actions on push to `main` using `eas update`.

The app targets both native and web (responsive) and uses the same backend (d-sports-api) as the PWA for API and checkout flows.
