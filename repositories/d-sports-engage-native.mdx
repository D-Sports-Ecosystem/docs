## Overview

**d-sports-engage-native** (package name: `engage-native`) is the native mobile app for D-Sports. It mirrors the core PWA experience on iOS and Android: wallet, shop, leaderboard, locker room, and profile.

- **Run:** `bunx expo start` or `bun run start` — then press `a` for Android or `i` for iOS, or scan the QR code with Expo Go.

## Features

- **Wallet** — Tokens, holdings, pack opening, crypto checkout (via PWA backend). Supports Arbitrum, Ethereum, and Polygon chains.
- **Shop** — Collectibles, cart, coin bundles, 3D model previews (Filament on native, Three.js on web), and crypto checkout.
- **Leaderboard** — Rankings and filters with modal detail views.
- **Locker room** — Social feed, team exploration, fan discovery, quests, and D-Sports Cash balance.
- **Games** — Daily Pick'Em, Daily Spin Wheel, and Guess the Player mini-games inside the locker room.
- **Teams** — Follow and fan teams, explore league standings, and view team detail modals.
- **Profile** — User profile, settings, and fan profile fields (favorite player, sport, team, rival).
- **Theme** — Dark/light mode (default dark) with brand colors (Gold `#F5C842`, Blue `#4169E1`).

## Getting started

<Steps>
  <Step title="Install dependencies">
    Clone the repository and run `bun install`.
  </Step>
  <Step title="Configure environment">
    The app requires these `EXPO_PUBLIC_*` environment variables:

    | Variable | Purpose |
    | -------- | ------- |
    | `EXPO_PUBLIC_CLERK_PUBLISHABLE_KEY` | Clerk authentication |
    | `EXPO_PUBLIC_API_URL` | Backend API base URL (`https://api.d-sports.org`) |
    | `EXPO_PUBLIC_TW_CLIENT_ID` | Thirdweb client ID for Web3 |
    | `EXPO_PUBLIC_REVENUECAT_API_KEY` | RevenueCat API key |
    | `EXPO_PUBLIC_REVENUECAT_APPSTORE_ID` | RevenueCat App Store ID |
    | `EXPO_PUBLIC_REVENUECAT_ENTITLEMENT` | RevenueCat entitlement name |
    | `EXPO_PUBLIC_SUPABASE_URL` | Supabase project URL |
    | `EXPO_PUBLIC_SUPABASE_KEY` | Supabase publishable key |

  </Step>
  <Step title="Start the dev server">
    Run `bunx expo start`, then press **a** for Android, **i** for iOS, or scan the QR code with Expo Go.
  </Step>
  <Step title="Create a development build (optional)">
    Use EAS to create native builds:

    ```bash
    bun run build:dev          # iOS + Android dev client
    bun run build:preview      # Internal distribution (APK on Android)
    bun run build:prod         # Production build
    ```

  </Step>
</Steps>

## API client

The app communicates with **d-sports-api** through a typed API client layer in `lib/api/`. All requests use the standard `{ success, data }` / `{ success: false, error, code? }` envelope.

You access API modules via the `useApi()` hook:

```typescript
import { useApi } from "@/lib/api";

const api = useApi();
const result = await api.teams.getTeams();
```

Available modules: `user`, `quests`, `leaderboard`, `wallet`, `lockerRoom`, `teams`, `collectibles`, `shop`, `checkout`.

The app targets both native and web (responsive) and uses the same backend (d-sports-api) as the PWA for API and checkout flows.
