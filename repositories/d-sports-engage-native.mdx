## Overview

**d-sports-engage-native** (package name: `engage-native`, version **1.14.5**) is the native mobile app for D-Sports. It mirrors the core PWA experience on iOS and Android: wallet, shop, leaderboard, locker room, and profile. The app also runs as a PWA on the web with a responsive desktop layout.

## Tech stack

| Category     | Technology                                        |
| ------------ | ------------------------------------------------- |
| Framework    | Expo 54, React Native 0.81, React 19              |
| Auth         | Clerk (`@clerk/clerk-expo`)                       |
| Payments     | RevenueCat (`react-native-purchases`)             |
| Web3         | Thirdweb SDK (Arbitrum, Ethereum, Polygon)        |
| State        | Zustand 5 with MMKV persistence                  |
| Storage      | MMKV (`react-native-mmkv`)                        |
| UI           | Lucide React Native                               |
| Navigation   | Expo Router 6 (file-based, typed routes)          |
| Animations   | React Native Reanimated 4                         |
| Monitoring   | Sentry (`@sentry/react-native`)                   |
| Notifications| OneSignal (`react-native-onesignal`)              |
| Package mgr  | Bun 1.3.9                                         |

## Features

- **Wallet** — tokens, holdings, pack opening, send/receive, crypto checkout (via PWA backend)
- **Shop** — collectibles, cart, coin bundles, 3D model viewer, checkout (fiat via RevenueCat, crypto via Thirdweb)
- **Leaderboard** — rankings and filters
- **Locker room** — social feed, daily pick-em, daily spin wheel, guess-the-player, quests, explore fans and teams
- **Profile** — user profile, achievements, followers/following, points history
- **Settings** — account, appearance, accessibility, notifications, privacy, data, audio, support, changelog
- **Onboarding** — new-user onboarding flow with team selection
- **Theme** — dark/light mode (default dark)

## Getting started

<Steps>
  <Step title="Install dependencies">
    ```bash
    bun install
    ```
  </Step>
  <Step title="Configure environment">
    Create a `.env` file with the following variables (only `EXPO_PUBLIC_*` keys are accessible at runtime):

    | Variable | Purpose |
    | -------- | ------- |
    | `EXPO_PUBLIC_CLERK_PUBLISHABLE_KEY` | Clerk authentication |
    | `EXPO_PUBLIC_API_URL` | API backend base URL |
    | `EXPO_PUBLIC_TW_CLIENT_ID` | Thirdweb client ID |
    | `EXPO_PUBLIC_REVENUECAT_API_KEY` | RevenueCat API key |
    | `EXPO_PUBLIC_REVENUECAT_APPSTORE_ID` | RevenueCat App Store ID |
    | `EXPO_PUBLIC_REVENUECAT_ENTITLEMENT` | RevenueCat entitlement (e.g. `premium`) |
    | `EXPO_PUBLIC_SUPABASE_URL` | Supabase project URL |
    | `EXPO_PUBLIC_SUPABASE_KEY` | Supabase publishable key |
  </Step>
  <Step title="Start development server">
    ```bash
    bunx expo start
    ```
    Press **a** for Android, **i** for iOS, or scan the QR code with Expo Go.
  </Step>
</Steps>

## Build and deploy

### EAS build profiles

| Profile        | Command                          | Distribution | Notes |
| -------------- | -------------------------------- | ------------ | ----- |
| Development    | `bun run build:dev`              | Internal     | Dev client, iOS simulator enabled |
| Preview        | `bun run build:preview`          | Internal     | Android APK |
| Staging        | EAS CLI                          | Auto-increment | Staging channel |
| Production     | `bun run build:prod`             | Auto-increment | Production channel |

You can also build per-platform with `bun run build:dev:ios`, `bun run build:dev:android`, etc.

A GitHub Actions workflow (`eas-update.yml`) publishes over-the-air updates via EAS Update on every push to the `master` branch.

## Architecture patterns

- **File-based routing** via Expo Router with route groups `(tabs)`, `(auth)`, `(onboarding)` and typed routes enabled
- **Modular screen architecture** — screen files contain only JSX; all state, effects, and handlers live in dedicated hooks (e.g. `use-wallet-screen.ts`, `use-shop-screen.ts`)
- **Extracted sub-components** — wallet and shop screens decomposed into `components/wallet/` and `components/shop/` with barrel exports
- **Zustand + MMKV** for global state with synchronous persistence
- **React Context** for auth, collectibles, navbar visibility, and accessibility settings
- **API client layer** in `lib/api/` with MMKV cache fallback (`lib/api/cache.ts`)
- **Dual checkout** — fiat via RevenueCat (Apple IAP, Google Play, Stripe on web) and crypto via Thirdweb SDK on Arbitrum, Ethereum, or Polygon
- **PWA-ready** — `display: standalone`, responsive desktop layout (`maxWidth: 480px`), web hover states, keyboard navigation
- **Path alias** `@/*` maps to the project root
