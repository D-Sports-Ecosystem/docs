## Overview

**d-sports-engage-native** (package name: `engage-native`) is the native mobile app for D-Sports. It mirrors the core PWA experience on iOS and Android: wallet, shop, leaderboard, locker room, and profile.

- **Run:** `bunx expo start` or `bun run start` — then press `a` for Android or `i` for iOS, or scan the QR code with Expo Go.

## Tech stack

| Category   | Technology                |
| ---------- | ------------------------- |
| Framework  | Expo 54, React Native 0.81, React 19 |
| Auth       | Clerk (Expo)              |

| Web3       | Thirdweb                  |

## Features

- **Wallet** — Tokens, holdings, pack opening, crypto checkout (via PWA backend)
- **Shop** — Collectibles, cart, coin bundles, checkout
- **3D collectible viewer** — Interactive GLB model viewer on web (orbit controls, auto-fit); native platforms render a placeholder
- **Leaderboard** — Rankings and filters
- **Locker room** — Social feed and engagement
- **Profile** — User profile and settings
- **Theme** — Dark/light mode (default dark)

- Quest UI now reflects team-scoped eligibility and pass-gated progression where applicable.
- Reward presentation differentiates free/unpaid states and claimability.
- Native progression state is expected to mirror backend eligibility and completion semantics.

## Team-aware experiences

- Team discovery, favorite-team interactions, and leaderboard context are integrated across locker room and profile surfaces.

## 3D GLB model viewer

The shop uses a platform-split `WebGlbViewer` component to render interactive 3D collectible models:

- **Web** (`WebGlbViewer.web.tsx`) — uses `@react-three/fiber` Canvas with orbit controls, ambient and directional lighting, and auto-fit bounds. Models load via `useGLTF` with an error boundary fallback.
- **Native** (`WebGlbViewer.tsx`) — renders an empty placeholder since Three.js requires a WebGL context.

The viewer accepts a `modelUrl` (remote `.glb` file), an optional `viewerKey` to force re-renders, and `onLoad`/`onError` callbacks.

## CI/CD

An **EAS Update** workflow (`.github/workflows/eas-update.yml`) runs on pushes to `master` and publishes over-the-air updates:

- Installs dependencies with Bun, sets up Node 22 for EAS tooling, and authenticates via `EXPO_TOKEN`.
- Publishes to the `develop` branch channel on `develop` pushes and the `main` branch channel on `main` pushes.

## Web deployment

The web build deploys to **Vercel** using the Expo web export:

- **Build command:** `npx expo export --platform web`
- **Output directory:** `dist`
- **SPA routing** — all non-asset paths rewrite to `index.html`.
- **Image proxy** — `/api/image-proxy` rewrites to the API backend for same-origin Clerk image loading.
- **CSP headers** — allow `blob:`, `https:`, and `wss:` for Thirdweb, WalletConnect, and Clerk.

The app targets both native and web (responsive) and uses the same backend (d-sports-api) as the PWA for API and checkout flows.
