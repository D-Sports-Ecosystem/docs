## Overview

**d-sports-engage-native** (package name: `engage-native`, version `1.10.10`) is the native mobile app for D-Sports. It mirrors the core PWA experience on iOS and Android: wallet, shop, leaderboard, locker room, and profile. It also supports web via Expo's web bundler as a PWA.

## Tech stack

| Category   | Technology                                    |
| ---------- | --------------------------------------------- |
| Framework  | Expo 54, React Native 0.81, React 19          |
| Auth       | Clerk (Expo)                                  |
| Payments   | RevenueCat (react-native-purchases)           |
| Web3       | Thirdweb SDK                                  |
| State      | Zustand + MMKV persistence                    |
| Storage    | MMKV (synchronous)                            |
| UI         | Lucide React Native                           |
| Navigation | Expo Router 6 (file-based, typed routes)      |
| Monitoring | Sentry                                        |
| Package    | Bun                                           |

## Features

- **Wallet** — tokens, holdings, pack opening, crypto checkout (via PWA backend)
- **Shop** — collectibles, cart, coin bundles, checkout (fiat via RevenueCat, crypto via Thirdweb)
- **Leaderboard** — rankings and filters
- **Locker room** — social feed, quests, daily games (Pick 'Em, Spin Wheel, Guess the Player)
- **Profile** — user profile, settings, and team membership
- **Theme** — dark/light mode (default dark)
- **Onboarding** — guided new-user flow with team selection

## Getting started

### Prerequisites

- **Bun** installed on your machine
- An Apple or Android development environment (Xcode / Android Studio) for native builds
- Expo Go app for quick testing on a physical device

### Environment variables

Create a `.env` file in the project root with the following `EXPO_PUBLIC_*` keys:

| Variable                              | Description                       |
| ------------------------------------- | --------------------------------- |
| `EXPO_PUBLIC_CLERK_PUBLISHABLE_KEY`   | Clerk publishable key             |
| `EXPO_PUBLIC_API_URL`                 | API backend base URL              |
| `EXPO_PUBLIC_TW_CLIENT_ID`           | Thirdweb client ID                |
| `EXPO_PUBLIC_REVENUECAT_API_KEY`     | RevenueCat API key                |
| `EXPO_PUBLIC_REVENUECAT_APPSTORE_ID` | RevenueCat App Store ID           |
| `EXPO_PUBLIC_REVENUECAT_ENTITLEMENT` | RevenueCat entitlement identifier |
| `EXPO_PUBLIC_SUPABASE_URL`           | Supabase project URL              |
| `EXPO_PUBLIC_SUPABASE_KEY`           | Supabase publishable (anon) key   |

<Note>Only `EXPO_PUBLIC_*` keys are accessible at runtime in the Expo app.</Note>

### Run the app

```bash
# Install dependencies
bun install

# Start development server
bunx expo start
# Press 'a' for Android, 'i' for iOS, or scan QR with Expo Go
```

### EAS builds

The project uses EAS Build for native builds. Profiles are defined in `eas.json`:

```bash
# Development build (all platforms)
bun run build:dev

# Preview build
bun run build:preview

# Production build
bun run build:prod

# Platform-specific (append :ios or :android)
bun run build:dev:ios
bun run build:prod:android

# OTA update
bun run update

# Submit to app stores
bun run submit:ios
bun run submit:android
```

## Architecture patterns

### Screen pattern

Screen files (`wallet.tsx`, `shop.tsx`) contain **only JSX**. All state, effects, and handlers live in dedicated hooks:

- `hooks/use-wallet-screen.ts` — wallet/token fetching, PIN verification, transaction handlers
- `hooks/use-shop-screen.ts` — cart state, product queries, carousel auto-scroll, checkout logic

Sub-components are extracted into `components/wallet/` and `components/shop/` with barrel exports via `index.ts`.

### API client layer

- `lib/api/client.ts` — base HTTP client with Clerk auth token injection
- `lib/api/cache.ts` — `fetchWithCache()` using MMKV for offline-first data
- Domain modules: `wallet-api.ts`, `shop-api.ts`, `user-api.ts`, `quests-api.ts`, `leaderboard-api.ts`, `locker-room-api.ts`, `teams-api.ts`, `collectibles-api.ts`, `checkout-api.ts`

### Checkout and payments

- **RevenueCat** — fiat payments: Apple IAP (native), Google Play (native), Stripe (web)
- **Thirdweb SDK** — on-chain crypto payments via `prepareTransaction` + `sendTransaction`
- Supported chains: Arbitrum (default), Ethereum, Polygon

### Path alias

Import using the `@/*` alias (maps to project root):

```typescript
import { useUser } from "@/context/user-context";
import type { Token } from "@/types/wallet.types";
```

The app targets both native and web (responsive) and uses the same backend (**d-sports-api**) as the PWA for API and checkout flows.
