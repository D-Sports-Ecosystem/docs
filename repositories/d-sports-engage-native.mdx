## Overview

**d-sports-engage-native** (package name: `engage-native`) is the native mobile app for D-Sports. It mirrors the core PWA experience on iOS and Android: wallet, shop, leaderboard, locker room, and profile.

The app targets both native and web (responsive) and uses the same backend (`d-sports-api`) for API and checkout flows.

## Tech stack

| Category   | Technology                |
| ---------- | ------------------------- |
| Framework  | Expo 54, React Native 0.81, React 19 |
| Auth       | Clerk (`@clerk/clerk-expo`) |
| Payments   | RevenueCat (`react-native-purchases`) |
| Web3       | Thirdweb SDK              |
| State      | Zustand + MMKV persistence |
| Storage    | MMKV (synchronous)        |
| UI         | Lucide React Native       |
| Navigation | Expo Router 6 (file-based) |
| Monitoring | Sentry (`@sentry/react-native`) |
| Package    | Bun 1.3.9                 |

## Features

- **Wallet** — tokens, holdings, pack opening, crypto checkout (Arbitrum, Ethereum, Polygon)
- **Shop** — collectibles, cart, coin bundles, fiat and crypto checkout
- **Leaderboard** — rankings and filters
- **Locker room** — social feed, quests, daily games (Pick'Em, Spin Wheel, Guess the Player)
- **Profile** — user profile, settings, team membership
- **Theme** — dark/light mode (default dark)

## Getting started

    ```bash
    bun install
    ```
  </Step>
  <Step title="Configure environment variables">
    Create a `.env` file in the project root. The app reads `EXPO_PUBLIC_*` keys at runtime:

    | Variable | Description |
    | -------- | ----------- |
    | `EXPO_PUBLIC_CLERK_PUBLISHABLE_KEY` | Clerk publishable key for authentication |
    | `EXPO_PUBLIC_API_URL` | Base URL for the d-sports-api backend |
    | `EXPO_PUBLIC_TW_CLIENT_ID` | Thirdweb client ID for Web3 features |
    | `EXPO_PUBLIC_REVENUECAT_API_KEY` | RevenueCat API key for in-app purchases |
    | `EXPO_PUBLIC_REVENUECAT_APPSTORE_ID` | RevenueCat App Store ID |
    | `EXPO_PUBLIC_REVENUECAT_ENTITLEMENT` | RevenueCat entitlement identifier |
    | `EXPO_PUBLIC_SUPABASE_URL` | Supabase project URL |
    | `EXPO_PUBLIC_SUPABASE_KEY` | Supabase publishable (anon) key |

  <Step title="Start the development server">
    ```bash
    bunx expo start
    ```

## Architecture

### Routing

The app uses **Expo Router** with file-based routing and typed routes. Route groups separate concerns: `(tabs)` for main navigation, `(auth)` for login/signup, and `(onboarding)` for new users. Auth protection in `_layout.tsx` automatically redirects based on authentication and onboarding state.

### Screen pattern

Screen files contain only imports and JSX. All state, effects, and handlers live in dedicated hooks:

- `hooks/use-wallet-screen.ts` — wallet and token fetch, PIN verification, transaction handling
- `hooks/use-shop-screen.ts` — cart state, product queries, carousel auto-scroll, checkout logic

Sub-components are extracted into `components/wallet/` and `components/shop/` with barrel exports.

### State management

- **Zustand** store in `services/store.ts` with MMKV persistence for theme, cart, and points
- **React Context** providers for authentication (`UserContext`), collectibles (`CollectiblesContext`), and navbar visibility

### API client layer

The `lib/api/` directory contains domain-specific API modules (wallet, shop, user, quests, leaderboard, locker room, teams, collectibles, checkout). A shared `useApi()` hook exposes all modules. Requests include auth-token injection via `lib/api/client.ts`, and `lib/api/cache.ts` provides MMKV cache-first fetching.

### Payments

- **Fiat payments** — RevenueCat handles Apple IAP (native), Google Play (native), and Stripe (web) via `lib/revenuecat/provider.tsx`
- **Crypto checkout** — Thirdweb SDK signs on-chain transactions. The flow calls `POST /api/checkout/crypto` and `POST /api/checkout/crypto/verify` on the PWA backend. Supported chains: Arbitrum (default), Ethereum, Polygon.

## EAS builds and CI/CD

The project uses **EAS Build** for native builds with these profiles:

| Profile | Use case |
| ------- | -------- |
| `development` | Dev client builds with simulator support |
| `preview` | Internal distribution (APK on Android) |
| `staging` | Auto-incrementing staging builds |
| `production` | Auto-incrementing production builds |

Common build commands:

```bash
bun run build:dev           # Development build (all platforms)
bun run build:preview       # Preview/internal build (all platforms)
bun run build:prod          # Production build (all platforms)
bun run build:dev:ios       # Development build (iOS only)
bun run build:prod:android  # Production build (Android only)
```

A **GitHub Actions workflow** (`eas-update.yml`) publishes OTA updates via `eas update` when commits are pushed to `develop` or `main`.
