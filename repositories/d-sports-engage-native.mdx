## Overview

**d-sports-engage-native** (package name: `engage-native`) is the native mobile app for D-Sports. It mirrors the core PWA experience on iOS and Android: wallet, shop, leaderboard, locker room, and profile.

- **Run:** `bunx expo start` or `bun run start` — then press `a` for Android or `i` for iOS, or scan the QR code with Expo Go.

## Tech stack

| Category   | Technology                |
| ---------- | ------------------------- |
| Framework  | Expo 54, React Native 0.81, React 19 |
| Auth       | Clerk (Expo)              |
| Payments   | RevenueCat (react-native-purchases) |
| Web3       | Thirdweb                  |
| State      | Zustand                   |
| Storage    | MMKV                      |
| Database   | Supabase (client-side JS) |
| UI         | Lucide React Native       |
| Navigation | Expo Router 6             |
| Package    | Bun                       |

## Features

- **Wallet** — Tokens, holdings, pack opening, crypto checkout (via PWA backend)
- **Shop** — Collectibles, cart, coin bundles, checkout
- **Leaderboard** — Rankings and filters
- **Locker room** — Social feed and engagement
- **Profile** — User profile and settings
- **Favourites sync** — Team favourites sync across devices, sessions, and page refreshes via the API with MMKV local fallback
- **Theme** — Dark/light mode (default dark)

## Getting started

<Steps>
  <Step title="Install dependencies">
    Clone the repository and install packages:

    ```bash
    bun install
    ```
  </Step>
  <Step title="Configure environment variables">
    Create a `.env` file at the project root. Only `EXPO_PUBLIC_*` keys are accessible at runtime:

    | Variable | Purpose |
    | -------- | ------- |
    | `EXPO_PUBLIC_CLERK_PUBLISHABLE_KEY` | Clerk authentication (live instance) |
    | `EXPO_PUBLIC_API_URL` | Backend API base URL (e.g. `https://api.d-sports.org`) |
    | `EXPO_PUBLIC_TW_CLIENT_ID` | Thirdweb client ID for Web3 features |
    | `EXPO_PUBLIC_REVENUECAT_API_KEY` | RevenueCat API key for in-app purchases |
    | `EXPO_PUBLIC_REVENUECAT_APPSTORE_ID` | RevenueCat App Store identifier |
    | `EXPO_PUBLIC_REVENUECAT_ENTITLEMENT` | RevenueCat entitlement name (e.g. `premium`) |
    | `EXPO_PUBLIC_SUPABASE_URL` | Supabase project URL |
    | `EXPO_PUBLIC_SUPABASE_KEY` | Supabase publishable (anon) key |
  </Step>
  <Step title="Start the development server">
    ```bash
    bunx expo start
    ```

    Press `a` for Android, `i` for iOS, or scan the QR code with Expo Go.
  </Step>
  <Step title="Build for device (optional)">
    For development builds on a physical device, use EAS:

    ```bash
    bun run build:dev
    ```

    Or target a single platform with `bun run build:dev:ios` / `bun run build:dev:android`.
  </Step>
</Steps>

The app targets both native and web (responsive) and uses the same backend (d-sports-api) as the PWA for API and checkout flows.

## Architecture

- **File-based routing** via Expo Router with route groups `(tabs)`, `(auth)`, `(onboarding)`.
- **Modular screen architecture** — screen files contain only JSX; all state, effects, and handlers live in dedicated hooks (e.g. `use-wallet-screen.ts`).
- **API client layer** in `lib/api/` with MMKV cache fallback (`lib/api/cache.ts`) and a standardised `{ success, data }` / `{ success: false, error }` response envelope.
- **Optimistic updates** — actions like following a team update local state immediately and revert on API failure.
- **Zustand + MMKV** for global state with synchronous persistence.
- **React Context** for auth (`user-context`), collectibles, and navbar visibility.
- **Crypto checkout** via Thirdweb SDK, calling the PWA backend for on-chain payments.
- **Path alias** `@/*` maps to the project root.

## Favourites sync

Team favourites now sync across devices and sessions through the API. When you follow or unfollow a team, the app:

1. Applies an **optimistic update** to local state and MMKV storage immediately.
2. Calls the teams API (`POST /api/teams/{id}/follow` or `DELETE /api/teams/{id}/follow`).
3. Reverts local state if the API call fails.

On login or page refresh, the user profile is fetched from `GET /api/user` which includes `favoriteTeam`. Followed teams are enriched via `GET /api/teams/followed`. This ensures favourites stay consistent regardless of which device you use.

<Note>
  You can follow up to **8 teams**. The first team you follow becomes your primary `teamId`.
</Note>
