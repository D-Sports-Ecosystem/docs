## Overview

**d-sports-engage-native** (package name: `engage-native`, version **1.13.1**) is the native mobile app for D-Sports. It mirrors the core PWA experience on iOS and Android: wallet, shop, leaderboard, locker room, and profile. The app also supports web via Expo's static output.

## Tech stack

| Category    | Technology                                    |
| ----------- | --------------------------------------------- |
| Framework   | Expo 54, React Native 0.81, React 19          |
| Language    | TypeScript 5.9                                |
| Auth        | Clerk (Expo)                                  |
| Payments    | RevenueCat (react-native-purchases)           |
| Web3        | Thirdweb SDK                                  |
| Push        | OneSignal (onesignal-expo-plugin)             |
| State       | Zustand + MMKV persistence                    |
| Navigation  | Expo Router 6 (file-based, typed routes)      |
| UI          | Lucide React Native                           |
| Error tracking | Sentry (@sentry/react-native)             |
| Analytics   | Vercel Analytics                              |
| Package mgr | Bun                                           |

## Features

- **Wallet** — token balances, holdings, pack opening, crypto checkout (Arbitrum, Ethereum, Polygon via Thirdweb), external wallet provider support
- **Shop** — collectibles, cart, coin bundles, pack detail, crypto and fiat checkout (RevenueCat for Apple IAP / Google Play / Stripe)
- **Leaderboard** — rankings and filters
- **Locker room** — social feed, daily pick-em, daily spin wheel, quests, team exploration
- **Profile** — user profile editing, avatar/banner uploads, followers/following, points history
- **Settings** — API-backed preferences (appearance, notifications, privacy, accessibility, room, data), blocked users, in-app support tickets, data export, legal links
- **Push notifications** — OneSignal SDK with native subscription sync (iOS, Android, web)
- **Haptics** — tactile feedback on supported devices (tab bar, settings rows, controls), respects accessibility preferences
- **Theme** — dark/light mode (default dark), server-persisted preference

## Getting started

<Steps>
  <Step title="Clone and install">
    Clone the repository and install dependencies:
    ```bash
    git clone <repo-url>
    bun install
    ```
  </Step>
  <Step title="Configure environment">
    Create a `.env` file in the project root. Only `EXPO_PUBLIC_*` keys are accessible at runtime:

    | Variable | Description |
    | -------- | ----------- |
    | `EXPO_PUBLIC_CLERK_PUBLISHABLE_KEY` | Clerk publishable key for authentication |
    | `EXPO_PUBLIC_API_URL` | Backend API base URL (e.g. `https://api.d-sports.org`) |
    | `EXPO_PUBLIC_TW_CLIENT_ID` | Thirdweb client ID for web3 features |
    | `EXPO_PUBLIC_REVENUECAT_API_KEY` | RevenueCat API key for in-app purchases |
    | `EXPO_PUBLIC_REVENUECAT_APPSTORE_ID` | RevenueCat App Store app ID |
    | `EXPO_PUBLIC_REVENUECAT_ENTITLEMENT` | RevenueCat entitlement identifier (e.g. `premium`) |
    | `EXPO_PUBLIC_SUPABASE_URL` | Supabase project URL (for client-side features, not direct DB) |
    | `EXPO_PUBLIC_SUPABASE_KEY` | Supabase publishable anon key |
  </Step>
  <Step title="Start the dev server">
    ```bash
    bunx expo start
    ```
    Press `a` for Android, `i` for iOS, or scan the QR code with Expo Go.
  </Step>
</Steps>

## Build commands

```bash
# Development
bunx expo start              # Start dev server
bun tsc --noEmit             # TypeScript check

# EAS builds
bun run build:dev            # Development build (all platforms)
bun run build:dev:ios        # Development build (iOS simulator)
bun run build:dev:android    # Development build (Android)
bun run build:preview        # Preview / internal distribution
bun run build:prod           # Production build (all platforms)

# Web
bun run web                  # Start web dev server
bun run build:web            # Export static web build

# Native (local)
bun run local:android        # Run on local Android device
bun run local:ios            # Run on local iOS device

# Submit to stores
bun run submit:ios           # Submit to App Store
bun run submit:android       # Submit to Google Play
```

## Architecture patterns

- **Modular screens** — screen files contain only JSX; all state, effects, and handlers live in dedicated hooks (e.g. `use-wallet-screen.ts`, `use-shop-screen.ts`)
- **Extracted sub-components** — wallet and shop screens decompose into `components/wallet/` and `components/shop/` with barrel exports
- **API client layer** — `lib/api/client.ts` provides an authenticated HTTP client; domain modules handle wallet, shop, user, quests, and more; `lib/api/cache.ts` adds MMKV cache-first fetching
- **Dual checkout** — RevenueCat handles fiat payments (Apple IAP, Google Play, Stripe on web); Thirdweb SDK handles on-chain crypto payments via the PWA backend
- **Path alias** — use `@/*` to import from the project root (e.g. `import { useUser } from "@/context/user-context"`)
