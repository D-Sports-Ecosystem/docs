## Overview

**d-sports-engage-native** (package name: `engage-native`) is the native mobile app for D-Sports. It mirrors the core PWA experience on iOS and Android: wallet, shop, leaderboard, locker room, and profile.

Run `bunx expo start` — then press **a** for Android, **i** for iOS, or scan the QR code with Expo Go.

## Features

- **Wallet** — Tokens, holdings, pack opening, crypto checkout (via PWA backend)
- **Shop** — Collectible packs with search, rarity filters, featured carousel, cart, coin bundles, and checkout
- **Collectibles** — Inventory of owned collectibles and packs, pack opening with reveal animation, opened-pack history
- **Leaderboard** — Rankings and filters
- **Locker room** — Social feed, quests, daily games (Pick'Em, Spin Wheel, Guess the Player), and team exploration
- **Profile** — User profile, settings, and team membership
- **Theme** — Dark/light mode (default dark)

## Architecture

### Screen pattern

Screen files under `app/(tabs)/` contain only imports and JSX. All state, effects, and handlers live in dedicated hooks:

- `hooks/use-wallet-screen.ts` — wallet and token fetch, PIN verification, transaction handlers
- `hooks/use-shop-screen.ts` — cart state, product queries, carousel auto-scroll, checkout logic

Sub-components are extracted into `components/wallet/` and `components/shop/` with barrel exports via `index.ts`.

### Collectibles system

The collectibles system manages pack purchases, pack opening, and inventory tracking through the `CollectiblesContext` provider. It exposes:

- **Owned collectibles** — cards with rarity levels (Common, Rare, Epic, Legendary) fetched from `/api/user/collectibles`
- **Owned packs** — purchased but unopened packs from `/api/user/packs`
- **Opened packs** — history of previously opened packs from `/api/user/packs/opened`
- **Pack opening** — calls `/api/packs/{id}/open` and returns revealed collectible cards

The shop screen displays available packs with rarity filters (Legendary, Epic, Rare, Common), a featured-packs carousel, search, favorites, and a cart with D-Sports Cash or crypto checkout.

### Payments

- **Fiat** — RevenueCat handles Apple IAP (native), Google Play (native), and Stripe (web)
- **Crypto** — Thirdweb SDK signs on-chain transactions via the PWA backend (`POST /api/checkout/crypto` and `POST /api/checkout/crypto/verify`). Supported chains: Arbitrum (default), Ethereum, Polygon.

### State management

- **Zustand** store (`services/store.ts`) with MMKV persistence for theme, cart, and points
- **React Context** for auth/user profile (`UserContext`), collectibles inventory (`CollectiblesContext`), and navbar visibility
- **API cache** — `lib/api/cache.ts` provides MMKV-backed cache-first fetching

## Getting started

<Steps>
  <Step title="Install dependencies">
    ```bash
    bun install
    ```
  </Step>
  <Step title="Configure environment variables">
    Create a `.env` file with the following `EXPO_PUBLIC_*` keys:

    | Variable | Description |
    | -------- | ----------- |
    | `EXPO_PUBLIC_CLERK_PUBLISHABLE_KEY` | Clerk authentication publishable key |
    | `EXPO_PUBLIC_API_URL` | Backend API base URL (e.g. `https://api.d-sports.org`) |
    | `EXPO_PUBLIC_TW_CLIENT_ID` | Thirdweb client ID for Web3 |
    | `EXPO_PUBLIC_REVENUECAT_API_KEY` | RevenueCat API key |
    | `EXPO_PUBLIC_REVENUECAT_APPSTORE_ID` | RevenueCat App Store ID |
    | `EXPO_PUBLIC_REVENUECAT_ENTITLEMENT` | RevenueCat entitlement identifier |
    | `EXPO_PUBLIC_SUPABASE_URL` | Supabase project URL |
    | `EXPO_PUBLIC_SUPABASE_KEY` | Supabase publishable (anon) key |
  </Step>
  <Step title="Start the development server">
    ```bash
    bunx expo start
    ```
    Press **a** for Android, **i** for iOS, or scan the QR code with Expo Go.
  </Step>
</Steps>

## EAS build profiles

Builds use [Expo Application Services (EAS)](https://expo.dev/eas). Available profiles in `eas.json`:

| Profile | Channel | Use case |
| ------- | ------- | -------- |
| `development` | `development` | Dev client with simulator support |
| `development-device` | `development` | Dev client on physical device |
| `preview` | `preview` | Internal distribution (Android APK) |
| `staging` | `staging` | Pre-production with auto-increment |
| `production` | `production` | App Store / Play Store release |

```bash
# Development build (all platforms)
bun run build:dev

# Production build (iOS only)
bun run build:prod:ios
```

## CI/CD

A GitHub Actions workflow (`.github/workflows/eas-update.yml`) publishes OTA updates via `eas update` on pushes to `develop` and `main` branches. The workflow uses Bun 1.3.9 and Node 22.

The app targets both native and web (responsive) and uses the same backend (`d-sports-api`) as the PWA for API and checkout flows.
