## Overview

**d-sports-engage-native** (package name: `engage-native`, version 1.14.6) is the native mobile app for D-Sports. It mirrors the core PWA experience on iOS and Android: wallet, shop, leaderboard, locker room, and profile. The app also supports web via Metro bundler with a responsive desktop layout.

## Tech stack

| Category      | Technology                                |
| ------------- | ----------------------------------------- |
| Framework     | Expo 54, React Native 0.81, React 19     |
| Auth          | Clerk (`@clerk/clerk-expo`)               |
| Payments      | RevenueCat (Apple IAP, Google Play, Stripe on web) |
| Web3          | Thirdweb SDK (Arbitrum, Ethereum, Polygon)|
| State         | Zustand 5 with MMKV persistence          |
| Storage       | MMKV (synchronous)                        |
| Monitoring    | Sentry (`@sentry/react-native`)           |
| Push          | OneSignal (`onesignal-expo-plugin`)       |
| UI            | Lucide React Native                       |
| Navigation    | Expo Router 6 (file-based, typed routes)  |
| Package       | Bun                                       |

## Features

- **Wallet** — tokens, holdings, pack opening, send/receive, crypto checkout via the PWA backend
- **Shop** — collectibles, cart, coin bundles, 3D model viewer, crypto and fiat checkout
- **Leaderboard** — rankings with filters and modal detail views
- **Locker room** — social feed, quests, daily pick'em, spin wheel, guess-the-player games, team exploration
- **Profile** — user profile, achievements, followers/following, points history
- **Settings** — account, appearance, notifications, accessibility, privacy, data, support, and changelog
- **Theme** — dark/light mode (default dark, brand gold `#F5C842`)

## Getting started

<Steps>
  <Step title="Install dependencies">
    ```bash
    bun install
    ```
  </Step>
  <Step title="Configure environment variables">
    Create a `.env` file at the project root with the following `EXPO_PUBLIC_*` keys:

    | Variable | Description |
    | -------- | ----------- |
    | `EXPO_PUBLIC_CLERK_PUBLISHABLE_KEY` | Clerk publishable key for authentication |
    | `EXPO_PUBLIC_API_URL` | Backend API base URL (e.g. `https://api.d-sports.org`) |
    | `EXPO_PUBLIC_TW_CLIENT_ID` | Thirdweb client ID for Web3 features |
    | `EXPO_PUBLIC_REVENUECAT_API_KEY` | RevenueCat API key |
    | `EXPO_PUBLIC_REVENUECAT_APPSTORE_ID` | RevenueCat App Store ID |
    | `EXPO_PUBLIC_REVENUECAT_ENTITLEMENT` | RevenueCat entitlement identifier |
    | `EXPO_PUBLIC_SUPABASE_URL` | Supabase project URL |
    | `EXPO_PUBLIC_SUPABASE_KEY` | Supabase publishable (anon) key |

    <Note>Only `EXPO_PUBLIC_*` variables are accessible at runtime in the Expo app.</Note>
  </Step>
  <Step title="Start the development server">
    ```bash
    bunx expo start
    ```
    Press **a** for Android, **i** for iOS, or scan the QR code with Expo Go.
  </Step>
</Steps>

## Build and deploy

The project uses **EAS Build** with the following profiles defined in `eas.json`:

| Profile | Purpose | Command |
| ------- | ------- | ------- |
| `development` | Dev client (simulator) | `bun run build:dev` |
| `development-device` | Dev client (physical device) | `bunx eas-cli build --profile development-device` |
| `preview` | Internal distribution (APK on Android) | `bun run build:preview` |
| `staging` | Staging with auto-increment | `bunx eas-cli build --profile staging` |
| `production` | Production release | `bun run build:prod` |

Over-the-air updates run automatically on push to `main` via the **EAS Update** GitHub Actions workflow.

## Architecture

- **Modular screens** — screen files contain only JSX; all state and logic live in dedicated hooks (`use-wallet-screen.ts`, `use-shop-screen.ts`).
- **File-based routing** — Expo Router with route groups `(tabs)`, `(auth)`, `(onboarding)` and typed routes enabled.
- **API client layer** — `lib/api/client.ts` handles auth token injection; domain modules (`wallet-api.ts`, `shop-api.ts`, etc.) are exposed through the `useApi()` hook. MMKV cache fallback is provided by `lib/api/cache.ts`.
- **Dual checkout** — fiat payments via RevenueCat (Apple IAP / Google Play / Stripe) and crypto payments via Thirdweb SDK calling the PWA backend (`POST /api/checkout/crypto`).
- **PWA-ready** — `display: standalone`, responsive desktop layout (`AppScreen` max-width 480 px), web hover states, and keyboard navigation support.
- **Path alias** — import using `@/*` (maps to project root).
