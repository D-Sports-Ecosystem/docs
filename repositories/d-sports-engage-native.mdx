## Overview

**d-sports-engage-native** (package name: `engage-native`) is the native mobile app for D-Sports. It mirrors the core PWA experience on iOS and Android: wallet, shop, leaderboard, locker room, and profile. The app also supports web via Expo's web target, deployed as a PWA on Vercel.

## Tech stack

| Category    | Technology                                     |
| ----------- | ---------------------------------------------- |
| Framework   | Expo 54, React Native 0.81, React 19           |
| Auth        | Clerk (Expo)                                   |
| Payments    | RevenueCat (react-native-purchases)            |
| Web3        | Thirdweb                                       |
| State       | Zustand                                        |
| Storage     | MMKV                                           |
| UI          | Lucide React Native                            |
| Navigation  | Expo Router 6 (file-based routing)             |
| 3D Viewer   | Three.js + React Three Fiber (web), Filament (native) |
| Monitoring  | Sentry                                         |
| Package     | Bun                                            |

## Features

- **Wallet** — Tokens, holdings, pack opening, crypto checkout (via PWA backend)
- **Shop** — Collectibles, cart, coin bundles, checkout with 3D GLB model previews
- **Leaderboard** — Rankings and filters
- **Locker room** — Social feed, quests, daily games (Pick 'Em, Spin Wheel, Guess the Player)
- **Profile** — User profile, settings, and team membership
- **Theme** — Dark/light mode (default dark)
- **3D collectible viewer** — Interactive GLB model viewer with zoom/orbit controls on both web and native

├── locker-room/         # Locker room components (FeedSection, QuestSection, DailyPickEmGame, etc.)

├── ui/                  # Reusable UI primitives (Button, TextField, TutorialOverlay, etc.)
└── Icon/                # Icon wrapper using lucide-react-native

## Environment variables

The app uses `EXPO_PUBLIC_*` keys, which are accessible at runtime in the Expo app. Create a `.env` file in the repo root:

```bash
# Clerk authentication
EXPO_PUBLIC_CLERK_PUBLISHABLE_KEY="pk_live_..."

# API backend
EXPO_PUBLIC_API_URL="https://api.d-sports.org"

# Thirdweb (Web3)
EXPO_PUBLIC_TW_CLIENT_ID="your-thirdweb-client-id"

# RevenueCat (in-app purchases)
EXPO_PUBLIC_REVENUECAT_API_KEY="your-revenuecat-key"
EXPO_PUBLIC_REVENUECAT_APPSTORE_ID="your-appstore-id"
EXPO_PUBLIC_REVENUECAT_ENTITLEMENT="premium"

# Supabase (client-side features, not direct DB)
EXPO_PUBLIC_SUPABASE_URL="https://your-project.supabase.co"
EXPO_PUBLIC_SUPABASE_KEY="sb_publishable_..."
```

## Getting started

<Steps>
  <Step title="Install dependencies">
    Clone the repository and install with Bun:

    ```bash
    bun install
    ```
  </Step>
  <Step title="Configure environment">
    Copy the `.env` example above and fill in values for Clerk, RevenueCat, Thirdweb, Supabase, and the API base URL.
  </Step>
  <Step title="Start the dev server">
    ```bash
    bunx expo start
    ```

    Press `a` for Android, `i` for iOS, or scan the QR code with Expo Go.
  </Step>
  <Step title="Run on web">
    ```bash
    bun run web
    ```

    This starts the Expo web target using Metro bundler.
  </Step>
</Steps>

## Build and deploy

### EAS builds

Use EAS CLI through Bun for cloud builds:

| Command                        | Description                 |
| ------------------------------ | --------------------------- |
| `bun run build:dev`            | Development build (all platforms) |
| `bun run build:preview`        | Preview/TestFlight build    |
| `bun run build:prod`           | Production build            |
| `bun run build:dev:ios`        | Development build (iOS only)  |
| `bun run build:dev:android`    | Development build (Android only) |
| `bun run update`               | Push OTA update via EAS Update |
| `bun run submit`               | Submit to App Store and Play Store |

### Web deployment

The web target exports as a static site and deploys to Vercel. Run `bun run build:web` to generate the `dist/` output directory. The `vercel.json` config handles SPA routing and image proxy rewrites.

## Architecture patterns

- **File-based routing** via Expo Router with route groups `(tabs)`, `(auth)`, `(onboarding)`
- **Modular screen architecture** — screen files contain only JSX; all state, effects, and handlers live in dedicated hooks (e.g., `use-wallet-screen.ts`, `use-shop-screen.ts`)
- **Extracted sub-components** — wallet and shop screens are decomposed into focused components with barrel exports
- **API client layer** in `lib/api/` with MMKV cache fallback
- **Zustand + MMKV** for global state with synchronous persistence
- **React Context** for auth, collectibles, and navbar visibility
- **Platform-specific 3D viewer** — `WebGlbViewer` uses Three.js/React Three Fiber on web; `NativeGlbViewer` uses react-native-filament on native
- **Path alias** `@/*` maps to the project root
