## Overview

**d-sports-engage-native** (package name: `engage-native`) is the native mobile app for D-Sports. It mirrors the core PWA experience on iOS and Android: wallet, shop, leaderboard, locker room, and profile. The app also runs as a responsive web PWA.

- **Run:** `bunx expo start` — then press `a` for Android, `i` for iOS, or scan the QR code with Expo Go.

## Features

- **Wallet** — Tokens, holdings, pack opening with 3D physics, crypto checkout (via PWA backend)
- **Shop** — Collectibles, cart, coin bundles, crypto and fiat checkout
- **Leaderboard** — Rankings and filters
- **Locker room** — Social feed, quests, daily games (Pick'Em, Spin Wheel, Guess the Player)
- **Profile** — User profile, settings, and accessibility options
- **Theme** — Dark/light mode (default dark)
- **Pack opening** — 3D physics-based pack reveal with spinning, acceleration, and inertia (web and native)

## Build and deploy

The app uses **EAS Build** for native binaries and **EAS Update** for OTA updates.

| Command | Description |
| ------- | ----------- |
| `bun run build:dev` | Development build (all platforms) |
| `bun run build:preview` | Internal preview build |
| `bun run build:prod` | Production build |
| `bun run build:web` | Export for web deployment |
| `bun run update` | Push an OTA update via EAS Update |

EAS build profiles are defined in `eas.json` with channels for `development`, `preview`, `staging`, and `production`.

A GitHub Actions workflow (`.github/workflows/eas-update.yml`) automatically publishes OTA updates when you push to `main` or `develop`.

## Architecture

### Screen pattern

Screen files in `app/(tabs)/` contain **only JSX**. All state, effects, and handlers live in dedicated hooks:

- `hooks/use-wallet-screen.ts` — wallet and token logic, PIN verification, transactions
- `hooks/use-shop-screen.ts` — cart state, product queries, carousel, checkout

Sub-components are extracted into `components/wallet/` and `components/shop/` with barrel exports.

### API client

The `lib/api/` directory contains a modular API client with auth token injection via `lib/api/client.ts`. Domain-specific modules (`wallet-api.ts`, `shop-api.ts`, `user-api.ts`, etc.) are exposed through the `useApi()` hook in `lib/api/index.ts`. An MMKV-based cache layer (`lib/api/cache.ts`) provides offline-first data fetching.

### Checkout and payments

- **Fiat** — RevenueCat handles Apple IAP (native), Google Play (native), and Stripe (web) via `lib/revenuecat/provider.tsx`
- **Crypto** — Thirdweb SDK signs on-chain transactions; the PWA backend processes payments at `POST /api/checkout/crypto` and `POST /api/checkout/crypto/verify`. Supported chains: Arbitrum (default), Ethereum, Polygon.

### Path alias

Import using the `@/*` alias, which maps to the project root:

```typescript
import { useUser } from "@/context/user-context";
import { getTokenColors } from "@/lib/token-utils";
```
