## Overview

**d-sports-engage-native** (package name: `engage-native`) is the native mobile app for D-Sports. It mirrors the core PWA experience on iOS and Android: wallet, shop, leaderboard, locker room, and profile.

## Tech stack

| Category   | Technology                                    |
| ---------- | --------------------------------------------- |
| Framework  | Expo 54, React Native 0.81, React 19          |
| Language   | TypeScript 5.9                                |
| Auth       | Clerk (Expo)                                  |
| Payments   | RevenueCat (react-native-purchases)           |
| Web3       | Thirdweb                                      |
| State      | Zustand + MMKV                                |
| Animations | React Native Reanimated                       |
| UI         | Lucide React Native                           |
| Navigation | Expo Router 6 (file-based, typed routes)      |
| Push       | OneSignal                                     |
| Monitoring | Sentry                                        |
| Package    | Bun                                           |

## Features

- **Wallet** — tokens, holdings, pack opening, crypto checkout (via PWA backend on Arbitrum, Ethereum, and Polygon)
- **Shop** — collectibles, cart, coin bundles, fiat and crypto checkout
- **Leaderboard** — rankings and filters
- **Locker room** — social feed, quests, daily games (Pick'Em, Spin Wheel, Guess the Player)
- **Profile** — user profile, followers/following, points history
- **Settings** — account, notifications, accessibility, appearance, privacy, support
- **Theme** — dark/light mode (default dark)
- **PWA-ready** — responsive desktop layout (`maxWidth: 480px`), web hover states, keyboard navigation

## Getting started

<Steps>
  <Step title="Clone and install">
    ```bash
    git clone <repo-url>
    cd d-sports-engage-native
    bun install
    ```
  </Step>
  <Step title="Configure environment">
    The repo includes a `.env` file with the following `EXPO_PUBLIC_*` variables. Only `EXPO_PUBLIC_*` keys are accessible at runtime:

    | Variable | Purpose |
    | -------- | ------- |
    | `EXPO_PUBLIC_CLERK_PUBLISHABLE_KEY` | Clerk authentication |
    | `EXPO_PUBLIC_API_URL` | Backend API base URL (`https://api.d-sports.org`) |
    | `EXPO_PUBLIC_TW_CLIENT_ID` | Thirdweb Web3 client ID |
    | `EXPO_PUBLIC_REVENUECAT_API_KEY` | RevenueCat in-app purchases |
    | `EXPO_PUBLIC_REVENUECAT_APPSTORE_ID` | RevenueCat App Store ID |
    | `EXPO_PUBLIC_REVENUECAT_ENTITLEMENT` | RevenueCat entitlement name |
    | `EXPO_PUBLIC_SUPABASE_URL` | Supabase client URL |
    | `EXPO_PUBLIC_SUPABASE_KEY` | Supabase publishable key |
  </Step>
  <Step title="Start the dev server">
    ```bash
    bunx expo start
    ```
    Press **a** for Android, **i** for iOS, or scan the QR code with Expo Go.
  </Step>
  <Step title="Type-check">
    ```bash
    bun tsc --noEmit
    ```
  </Step>
</Steps>

## Architecture

### Screen pattern

Screen files in `app/(tabs)/` contain **only JSX**. All state, effects, and handlers live in dedicated hooks:

- `hooks/use-wallet-screen.ts` — wallet/token fetch, PIN verification, transaction handlers
- `hooks/use-shop-screen.ts` — cart state, product queries, carousel auto-scroll, checkout logic

Sub-components are extracted into `components/wallet/` and `components/shop/` with barrel exports via `index.ts`.

### State management

- **Zustand** store (`services/store.ts`) with MMKV persistence for theme, cart, and points
- **React Context** for auth (`UserContext`), collectibles (`CollectiblesContext`), and navbar visibility
- **MMKV** via `services/storage.ts` for synchronous local storage

### API layer

- `lib/api/client.ts` — base HTTP client with auth token injection
- `lib/api/index.ts` — `useApi()` hook exposing all API modules
- `lib/api/cache.ts` — `fetchWithCache()` MMKV cache-first fetching
- Domain modules: `wallet-api.ts`, `shop-api.ts`, `user-api.ts`, `quests-api.ts`, `leaderboard-api.ts`, `locker-room-api.ts`, `teams-api.ts`, `collectibles-api.ts`, `checkout-api.ts`, `games-api.ts`

### Payments

- **RevenueCat** (`lib/revenuecat/provider.tsx`) — fiat payments via Apple IAP (native), Google Play (native), and Stripe (web)
- **Crypto checkout** (`lib/api/checkout-api.ts`) — calls the PWA backend at `POST /api/checkout/crypto` and `POST /api/checkout/crypto/verify`
- **Thirdweb SDK** — on-chain transaction signing via `prepareTransaction` + `sendTransaction`
- Supported chains: Arbitrum (default), Ethereum, Polygon

### Path aliases

Import using the `@/*` alias (maps to project root):

```typescript
import { useUser } from "@/context/user-context";
import { getTokenColors } from "@/lib/token-utils";
import type { Token } from "@/types/wallet.types";
```

## EAS builds and deployments

The app uses Expo Application Services (EAS) for native builds, OTA updates, and store submissions.

### Build commands

| Command | Description |
| ------- | ----------- |
| `bun run build:dev` | Development build (both platforms) |
| `bun run build:dev:ios` | Development build (iOS simulator) |
| `bun run build:dev:android` | Development build (Android APK) |
| `bun run build:preview` | Preview/QA build (internal distribution) |
| `bun run build:prod` | Production build (store-ready) |

### OTA updates

Push JavaScript-only changes to existing installs without a new store build:

```bash
bun run update --branch production --message "Fix: wallet balance display"
```

| Build profile | Update channel | Who gets updates |
| ------------- | -------------- | ---------------- |
| `development` | `development`  | Dev builds       |
| `preview`     | `preview`      | Internal testers |
| `production`  | `production`   | App store users  |

<Warning>
OTA updates only work if the JavaScript is compatible with the installed native binary. If you add or change native modules, you must do a full build first and bump the `version` in `app.json`.
</Warning>

### Store submission

```bash
bun run submit        # Both platforms
bun run submit:ios    # iOS only
bun run submit:android # Android only
```
