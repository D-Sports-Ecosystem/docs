## Overview

**d-sports-engage-native** (package name: `engage-native`, current version **v1.17.2**) is the native mobile app for D-Sports. It mirrors the core PWA experience on iOS and Android: wallet, shop, leaderboard, locker room, and profile.

- **Run:** `bunx expo start` or `bun run start` — then press `a` for Android or `i` for iOS, or scan the QR code with Expo Go.

## Features

- **Wallet** — Tokens, holdings, pack opening, crypto checkout (via PWA backend)
- **Shop** — Collectibles, cart, coin bundles, checkout
- **Leaderboard** — Per-team and global rankings with stack routes
- **Locker room** — Social feed, quest center, daily spin, and guess-the-player games
- **Profile** — User profile with milestone avatar rings, achievement badges, and settings
- **Theme** — Dark/light mode (default dark)

## What changed in v1.17.2

- **Milestone avatar rings** — Profile photos display a tier-colored SVG ring and glow based on reward milestones (Starter through Legend). Users with fewer than two claimed rewards keep the brand gold ring (`#F5C842`).
- **Tier preview dots** — Tap unlocked milestone dots under the avatar to preview that tier's ring. Long-press a dot for a modal with the tier title and description. Selection is persisted per user via MMKV (native) and web storage.
- **`claimedRewardsCount` API field** — `getUser` responses now include a numeric count of claimed rewards so the client does not need to derive milestones from the `userRewards` array.
- **Shared milestone data** — Quest center milestone list imports shared `REWARD_MILESTONES` from `lib/reward-milestones.ts` so thresholds stay aligned with the avatar ring.

### Previous notable changes

- Per-team leaderboard stack routes (`/leaderboard/[teamSlug]`) and MMKV cache fix (v1.17.1).
- Locker room games as dedicated routes, points history endpoint, and quest cache improvements (v1.17.0).
- Remote profile badge artwork from Supabase with native disk cache (v1.16.0).
- Team-scoped quests, favorites gating (max-5), and isolated team leaderboard scoring (v1.14.0).
- Pack opening odds disclosure and accessibility color-blindness mode (v1.14.7).

## Backend integration surfaces

- Primary API source: `d-sports-api` under `/api/*`.
- Native app consumes team, league, quest, reward, wallet, commerce, and social endpoints.
- Retry/caching logic and auth token injection are centralized in `lib/api/client.ts`.
- Profile payloads include `claimedRewardsCount` and optional `badgeMetadata` for milestone and badge rendering.

## Quest and pass UX integration

- Quest UI reflects team-scoped eligibility and pass-gated progression where applicable.
- Reward presentation differentiates free/unpaid states and claimability.
- Native progression state mirrors backend eligibility and completion semantics.

## Team-aware experiences

- Team discovery, favorite-team interactions, and leaderboard context are integrated across locker room and profile surfaces.
- Team and league metadata is sourced from backend sync pipelines fed by the `leagues` repository.

## Compliance and legal: pack opening odds

- Native odds disclosure behavior is documented with policy-aligned copy and evidence requirements.
- Disclosure values must come from backend payloads and remain visible/accessible near open actions.

## Getting started

<Steps>
  <Step title="Install dependencies">
    ```bash
    bun install
    ```
  </Step>
  <Step title="Configure environment variables">
    Create a `.env` file in the project root. Only `EXPO_PUBLIC_*` keys are accessible at runtime.

    | Variable | Purpose |
    | -------- | ------- |
    | `EXPO_PUBLIC_CLERK_PUBLISHABLE_KEY` | Clerk authentication |
    | `EXPO_PUBLIC_API_URL` | Backend API base URL (e.g. `https://api.d-sports.org`) |
    | `EXPO_PUBLIC_TW_CLIENT_ID` | Thirdweb client ID for Web3 |
    | `EXPO_PUBLIC_REVENUECAT_API_KEY` | RevenueCat in-app purchases |
    | `EXPO_PUBLIC_REVENUECAT_APPSTORE_ID` | RevenueCat App Store ID |
    | `EXPO_PUBLIC_REVENUECAT_ENTITLEMENT` | RevenueCat entitlement identifier |
    | `EXPO_PUBLIC_SUPABASE_URL` | Supabase project URL (badge assets, storage) |
    | `EXPO_PUBLIC_SUPABASE_KEY` | Supabase publishable key |
  </Step>
  <Step title="Start the development server">
    ```bash
    bunx expo start
    ```
    Press **a** for Android, **i** for iOS, or scan the QR code with Expo Go.
  </Step>
  <Step title="Run native or EAS builds (optional)">
    ```bash
    # Development build (EAS)
    bun run build:dev

    # Preview build (internal distribution)
    bun run build:preview

    # Production build
    bun run build:prod

    # TypeScript check
    bun tsc --noEmit
    ```
  </Step>
</Steps>

The app targets both native and web (responsive) and uses the same backend (`d-sports-api`) as the PWA for API and checkout flows.
