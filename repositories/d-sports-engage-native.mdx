## Overview

**d-sports-engage-native** (package name: `engage-native`) is the native mobile app for D-Sports. It mirrors the core PWA experience on iOS and Android: wallet, shop, leaderboard, locker room, and profile.

- **Run:** `bunx expo start` or `bun run start` — then press `a` for Android or `i` for iOS, or scan the QR code with Expo Go.

## Features

- **Wallet** — Tokens, holdings, pack opening, crypto checkout (via PWA backend)
- **Shop** — Collectibles, cart, coin bundles, checkout
- **Leaderboard** — Rankings and filters
- **Locker room** — Social feed, discover teams, explore fans, daily games (Pick'Em, Spin Wheel, Guess the Player), and quests
- **Profile** — User profile, achievements, followers, and settings
- **Theme** — Dark/light mode (default dark)

### Discover teams

The locker room includes a **Discover Teams** section that fetches teams from `GET /api/teams` and displays them in a horizontally scrollable list. You can follow or unfollow teams directly from the cards. A **Top Teams** carousel highlights the top 10 ranked teams with fan counts and rank badges.

Teams are categorized as **Official** or **Community**, and the follow state syncs with your user profile via `POST /api/teams/[id]/follow` and `DELETE /api/teams/[id]/follow`.

## Architecture

- **File-based routing** via Expo Router with route groups `(tabs)`, `(auth)`, `(onboarding)`
- **Modular screen architecture** — screen files contain only JSX; all state, effects, and handlers live in dedicated hooks (e.g., `use-wallet-screen.ts`, `use-shop-screen.ts`)
- **API client layer** in `lib/api/` with domain modules (teams, wallet, shop, user, quests, etc.) and MMKV cache fallback
- **Zustand + MMKV** for global state with synchronous persistence
- **React Context** for auth, collectibles, navbar visibility, and accessibility settings
- **Crypto checkout** via Thirdweb SDK calling the PWA backend for on-chain payments (Arbitrum, Ethereum, Polygon)
- **PWA-ready** — responsive desktop layout (`maxWidth: 480px`), web hover states, keyboard navigation
- **Path alias** `@/*` maps to project root

## Getting started

<Steps>
  <Step title="Install dependencies">
    ```bash
    bun install
    ```
  </Step>
  <Step title="Configure environment variables">
    Create a `.env` file at the project root with the following keys:

    | Variable | Description |
    | -------- | ----------- |
    | `EXPO_PUBLIC_CLERK_PUBLISHABLE_KEY` | Clerk authentication publishable key |
    | `EXPO_PUBLIC_API_URL` | API backend URL (e.g., `https://api.d-sports.org`) |
    | `EXPO_PUBLIC_TW_CLIENT_ID` | Thirdweb client ID for Web3 features |
    | `EXPO_PUBLIC_REVENUECAT_API_KEY` | RevenueCat API key for in-app purchases |
    | `EXPO_PUBLIC_REVENUECAT_APPSTORE_ID` | RevenueCat App Store ID |
    | `EXPO_PUBLIC_REVENUECAT_ENTITLEMENT` | RevenueCat entitlement identifier |
    | `EXPO_PUBLIC_SUPABASE_URL` | Supabase project URL |
    | `EXPO_PUBLIC_SUPABASE_KEY` | Supabase publishable (anon) key |

    <Note>Only `EXPO_PUBLIC_*` keys are accessible at runtime in the Expo app.</Note>
  </Step>
  <Step title="Start the development server">
    ```bash
    bunx expo start
    ```
    Press `a` for Android, `i` for iOS, or scan the QR code with Expo Go.
  </Step>
  <Step title="Build for device (optional)">
    For development builds using EAS:

    ```bash
    bun run build:dev          # All platforms
    bun run build:dev:ios      # iOS only
    bun run build:dev:android  # Android only
    ```
  </Step>
</Steps>

The app targets both native and web (responsive) and uses the same backend (d-sports-api) as the PWA for API and checkout flows.
