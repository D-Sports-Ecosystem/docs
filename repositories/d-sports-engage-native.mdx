## Overview

**d-sports-engage-native** (package name: `engage-native`) is the native mobile app for D-Sports. It mirrors the core PWA experience on iOS and Android: wallet, shop, leaderboard, locker room, and profile.

- **Run:** `bunx expo start` or `bun run start` — then press `a` for Android or `i` for iOS, or scan the QR code with Expo Go.

## Features

- **Wallet** — Tokens, holdings, pack opening, crypto checkout (via PWA backend)
- **Shop** — Collectibles, cart, coin bundles, checkout
- **Leaderboard** — Rankings and filters
- **Locker room** — Social feed, quests, and featured games
- **Profile** — User profile and settings
- **Theme** — Dark/light mode (default dark)

### Games

The locker room includes a **Featured Games Carousel** with interactive mini-games that award Points:

| Game | Description |
| ---- | ----------- |
| **Prize Wheel Spin** | Daily free spins (5 per day, reset at midnight). Land on hockey gear to build a weekly collection or land on the coin segment for **150 Points** instantly. Complete the weekly set of 4 items before Sunday reset for a **500 Points** bonus. After each spin, you are prompted to confirm point redemption. |
| **Guess the Player** | A clue-based guessing game — clues reveal progressively (position, division, height, weight, jersey number, nationality). You have 5 guesses per round. |

The games use the `/api/games/daily-spin/reward` endpoint to award points. The reward payload includes the reward type (`coin` or `weekly_set`), the landed segment, and an optional team/leaderboard context.

### Quests

The **Quest Section** lets users complete multi-step quests scoped to global, team, or event categories. Quests are fetched from `/api/quests` and support daily check-in tracking, step completion, and reward claiming.

## Environment variables

All runtime variables use the `EXPO_PUBLIC_` prefix. Create a `.env` file in the repo root:

| Variable | Purpose |
| -------- | ------- |
| `EXPO_PUBLIC_CLERK_PUBLISHABLE_KEY` | Clerk authentication (live key) |
| `EXPO_PUBLIC_API_URL` | API backend base URL (e.g. `https://api.d-sports.org`) |
| `EXPO_PUBLIC_TW_CLIENT_ID` | Thirdweb client ID for Web3 features |
| `EXPO_PUBLIC_REVENUECAT_API_KEY` | RevenueCat API key for in-app purchases |
| `EXPO_PUBLIC_REVENUECAT_APPSTORE_ID` | RevenueCat App Store ID |
| `EXPO_PUBLIC_REVENUECAT_ENTITLEMENT` | RevenueCat entitlement name (e.g. `premium`) |
| `EXPO_PUBLIC_SUPABASE_URL` | Supabase project URL |
| `EXPO_PUBLIC_SUPABASE_KEY` | Supabase publishable key |

## Getting started

1. Clone the repository and run `bun install`.
2. Copy `.env.example` to `.env` and fill in the values above.
3. Run `bunx expo start`.
4. For development builds: `bun run build:dev` (EAS) or run with Expo dev client.

The app targets both native and web (responsive) and uses the same backend (d-sports-api) as the PWA for API and checkout flows.
