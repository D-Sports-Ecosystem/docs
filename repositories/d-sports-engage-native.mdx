## Overview

**d-sports-engage-native** (package name: `engage-native`) is the native mobile app for D-Sports. It mirrors the core PWA experience on iOS and Android: wallet, shop, leaderboard, locker room, and profile. The app uses the **Poppins** font family as the default typeface, consistent with the D-Sports ecosystem.

## Tech stack

## Features

- **Wallet** — Tokens, holdings, pack opening, crypto checkout (via PWA backend)
- **Shop** — Collectibles, cart, coin bundles, checkout with RevenueCat (fiat) and Thirdweb (crypto)
- **Leaderboard** — Rankings and filters
- **Locker room** — Social feed, quests, daily games (Pick'Em, spin wheel, guess the player)
- **Profile** — User profile with picture/banner upload, achievements modal, followers/following, and points history
- **Settings** — Account, appearance, notifications, privacy, accessibility, and support
- **Theme** — Dark/light mode (default dark)
- **Push notifications** — OneSignal integration for real-time alerts

## Getting started

<Steps>
  <Step title="Clone and install">
    Clone the repository and install dependencies:

    ```bash
    git clone <repo-url>
    cd d-sports-engage-native
    bun install
    ```
  </Step>
  <Step title="Configure environment variables">
    Create a `.env` file in the project root. Only `EXPO_PUBLIC_*` keys are accessible at runtime in the Expo app:

    | Variable | Description |
    | -------- | ----------- |
    | `EXPO_PUBLIC_CLERK_PUBLISHABLE_KEY` | Clerk authentication publishable key |
    | `EXPO_PUBLIC_API_URL` | Backend API base URL (e.g. `https://api.d-sports.org`) |
    | `EXPO_PUBLIC_TW_CLIENT_ID` | Thirdweb client ID for Web3 features |
    | `EXPO_PUBLIC_REVENUECAT_API_KEY` | RevenueCat API key for in-app purchases |
    | `EXPO_PUBLIC_REVENUECAT_APPSTORE_ID` | RevenueCat App Store ID |
    | `EXPO_PUBLIC_REVENUECAT_ENTITLEMENT` | RevenueCat entitlement identifier (e.g. `premium`) |
    | `EXPO_PUBLIC_SUPABASE_URL` | Supabase project URL |
    | `EXPO_PUBLIC_SUPABASE_KEY` | Supabase publishable (anon) key |
  </Step>
  <Step title="Start the development server">
    ```bash
    bunx expo start
    ```

    Press `a` for Android, `i` for iOS, or scan the QR code with Expo Go.
  </Step>
</Steps>

## EAS build profiles

The app uses [EAS Build](https://docs.expo.dev/build/introduction/) for native builds. Available profiles are configured in `eas.json`:

| Profile | Command | Description |
| ------- | ------- | ----------- |
| `development` | `bun run build:dev` | Dev client build with simulator support (iOS) |
| `development-device` | `bunx eas-cli build --profile development-device` | Dev client for physical devices |
| `preview` | `bun run build:preview` | Internal distribution, APK on Android |
| `staging` | `bunx eas-cli build --profile staging` | Auto-incrementing version, staging channel |
| `production` | `bun run build:prod` | Production build with auto-incrementing version |

To push an OTA update: `bun run update`.

## Architecture

The app follows a **modular screen architecture**: screen files contain only JSX, while all state, effects, and handlers live in dedicated hooks (e.g. `hooks/use-wallet-screen.ts`). Sub-components are extracted into `components/<domain>/` directories with barrel exports.

- **API client** — `lib/api/client.ts` provides a base HTTP client with Clerk auth token injection. Domain modules (`wallet-api.ts`, `shop-api.ts`, `user-api.ts`, etc.) are exposed through the `useApi()` hook in `lib/api/index.ts`.
- **Caching** — `lib/api/cache.ts` provides MMKV-based cache-first fetching.
- **Crypto checkout** — Thirdweb SDK for on-chain transaction signing, calling the PWA backend at `POST /api/checkout/crypto` and `POST /api/checkout/crypto/verify`. Supported chains: Arbitrum (default), Ethereum, Polygon.
- **Path alias** — Import using `@/*` which maps to the project root.
