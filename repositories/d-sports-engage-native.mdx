## Overview

**d-sports-engage-native** (package name: `engage-native`) is the native mobile app for D-Sports. It mirrors the core PWA experience on iOS and Android: wallet, shop, leaderboard, locker room, and profile.

- **Run:** `bunx expo start` or `bun run start` — then press `a` for Android or `i` for iOS, or scan the QR code with Expo Go.

## Tech stack

| Category   | Technology                |
| ---------- | ------------------------- |
| Framework  | Expo 54, React Native 0.81, React 19 |
| Auth       | Clerk (Expo)              |
| Payments   | RevenueCat (react-native-purchases) |
| Web3       | Thirdweb                  |
| Database   | Supabase (client-side, for Realtime/Storage) |
| State      | Zustand                   |
| Storage    | MMKV                      |
| Monitoring | Sentry                    |
| UI         | Lucide React Native       |
| Navigation | Expo Router 6             |
| Package    | Bun                       |

## Features

- **Wallet** — Real BIP39 wallet generation, tokens, holdings, pack opening, crypto checkout (via PWA backend), PIN-protected seed phrase reveal
- **Shop** — Collectibles, cart, coin bundles, D-Sports Cash payments, crypto checkout (Arbitrum, Ethereum, Polygon)
- **Leaderboard** — Rankings, filters, team-themed leaderboards with accent colors
- **Locker room** — Social feed, quests, daily games (Pick'Em, Spin Wheel, Guess the Player), D-Sports Cash, explore fans and teams
- **Profile** — User profile, settings, achievement badges
- **Theme** — Dark/light mode (default dark)

## Environment variables

All runtime variables use the `EXPO_PUBLIC_` prefix. Create a `.env` file at the repo root:

| Variable | Purpose |
| -------- | ------- |
| `EXPO_PUBLIC_CLERK_PUBLISHABLE_KEY` | Clerk authentication publishable key |
| `EXPO_PUBLIC_API_URL` | Backend API base URL (e.g. `https://api.d-sports.org`) |
| `EXPO_PUBLIC_TW_CLIENT_ID` | Thirdweb client ID for Web3 features |
| `EXPO_PUBLIC_REVENUECAT_API_KEY` | RevenueCat API key for in-app purchases |
| `EXPO_PUBLIC_REVENUECAT_APPSTORE_ID` | RevenueCat App Store app ID |
| `EXPO_PUBLIC_REVENUECAT_ENTITLEMENT` | RevenueCat entitlement identifier (e.g. `premium`) |
| `EXPO_PUBLIC_SUPABASE_URL` | Supabase project URL |
| `EXPO_PUBLIC_SUPABASE_KEY` | Supabase publishable (anon) key |

## Getting started

<Steps>
  <Step title="Install dependencies">
    ```bash
    bun install
    ```
  </Step>
  <Step title="Configure environment">
    Copy the `.env` file and fill in your keys for Clerk, RevenueCat, Thirdweb, Supabase, and the API URL.
  </Step>
  <Step title="Start the dev server">
    ```bash
    bunx expo start
    ```
    Press `a` for Android, `i` for iOS, or scan the QR code with Expo Go.
  </Step>
  <Step title="Run a development build (optional)">
    For native code features (Thirdweb adapter, MMKV, etc.), create a dev build with EAS:
    ```bash
    bun run build:dev
    ```
  </Step>
</Steps>

The app targets both native and web (responsive) and uses the same backend (d-sports-api) as the PWA for API and checkout flows.

## EAS build profiles

| Profile | Description |
| ------- | ----------- |
| `development` | Dev client with internal distribution (iOS simulator) |
| `development-device` | Dev client for physical iOS devices |
| `preview` | Internal distribution, generates Android APK |
| `staging` | Auto-increments version for staging tests |
| `production` | Auto-increments version for App Store / Google Play submission |

**Common commands:**

```bash
bun run build:dev          # Development build (EAS)
bun run build:preview      # Preview build (EAS)
bun run build:prod         # Production build (EAS)
bun run update             # Push an OTA update via EAS Update
bun run submit:ios         # Submit to Apple App Store
bun run submit:android     # Submit to Google Play
```

## Project structure

```
app/
├── (auth)/              # Login, sign-up, SSO callback
├── (onboarding)/        # New user onboarding flow
├── (tabs)/              # Main tab navigation (wallet, shop, leaderboard, locker room, profile)
├── settings/            # Settings pages with nested modals and tabs
└── _layout.tsx          # Root layout with providers and auth protection

components/
├── wallet/              # Wallet sub-components (TabButton, TokenRow, ActionModal, etc.)
├── shop/                # Shop sub-components (CartModal, CryptoCheckoutModal, etc.)
├── locker-room/         # Locker room components (feed, quests, games, explore)
├── leaderboard/         # Leaderboard components
├── settings/            # Settings components, modals, and tabs
├── layout/              # AppScreen responsive wrapper
├── ui/                  # Reusable UI primitives (Button, TextField, WebHoverWrapper)
└── Icon/                # Icon wrapper using lucide-react-native

hooks/                   # Custom hooks (use-wallet-screen, use-shop-screen, etc.)
lib/api/                 # API client modules (wallet, shop, user, quests, checkout, etc.)
context/                 # React Context providers (user, collectibles, navbar, actions)
types/                   # Shared TypeScript types per domain
constants/               # Shared constants per domain
theme/                   # Color system and design tokens
services/                # Zustand store, MMKV storage adapter, core types
```

<Note>
  Screen files in `app/(tabs)/` contain **only imports and JSX**. All state, effects, and handlers live in dedicated hooks under `hooks/` (e.g. `hooks/use-wallet-screen.ts`).
</Note>
