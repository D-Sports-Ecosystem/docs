## Overview

**d-sports-engage-native** (package name: `engage-native`) is the native mobile app for D-Sports. It mirrors the core PWA experience on iOS and Android: wallet, shop, leaderboard, locker room, and profile.

- **Run:** `bunx expo start` or `bun run start` — then press `a` for Android or `i` for iOS, or scan the QR code with Expo Go.

## Features

- **Wallet** — Tokens, holdings, pack opening, crypto checkout (via PWA backend)
- **Shop** — Featured pack carousel, collectibles, cart, coin bundles, D-Sports Cash and crypto checkout
- **Leaderboard** — Rankings and filters
- **Locker room** — Social feed, quests, daily games (Pick'Em, Spin Wheel, Guess the Player)
- **Profile** — User profile and settings
- **Theme** — Dark/light mode (default dark)

### Featured pack tiers

The shop displays a featured carousel with three pack tiers in a fixed order:

1. **Free Pack** — no cost, available to all users
2. **Standard Pack** — mid-tier pack
3. **Starter Pack** — introductory paid pack

The carousel is driven by `FEATURED_PACK_ORDER` in `hooks/use-shop-screen.ts`. The app matches products from the API by name, falling back to the API's `featured` flag if none of the expected names are found.

## Getting started

<Steps>
  <Step title="Install dependencies">
    Clone the repository and run `bun install`.
  </Step>
  <Step title="Configure environment variables">
    Create a `.env` file at the project root with the following `EXPO_PUBLIC_*` keys:

    | Variable | Purpose |
    | -------- | ------- |
    | `EXPO_PUBLIC_CLERK_PUBLISHABLE_KEY` | Clerk authentication |
    | `EXPO_PUBLIC_API_URL` | Backend API base URL (e.g. `https://api.d-sports.org`) |
    | `EXPO_PUBLIC_TW_CLIENT_ID` | Thirdweb client ID for Web3 |
    | `EXPO_PUBLIC_REVENUECAT_API_KEY` | RevenueCat API key |
    | `EXPO_PUBLIC_REVENUECAT_APPSTORE_ID` | RevenueCat App Store ID |
    | `EXPO_PUBLIC_REVENUECAT_ENTITLEMENT` | RevenueCat entitlement identifier |
    | `EXPO_PUBLIC_SUPABASE_URL` | Supabase project URL |
    | `EXPO_PUBLIC_SUPABASE_KEY` | Supabase publishable key |
  </Step>
  <Step title="Start the dev server">
    Run `bunx expo start`, then press **a** for Android, **i** for iOS, or scan the QR code with Expo Go.
  </Step>
  <Step title="Create a development build (optional)">
    Run `bun run build:dev` via EAS or use the Expo dev client for native modules.
  </Step>
</Steps>

The app targets both native and web (responsive) and uses the same backend (d-sports-api) as the PWA for API and checkout flows.

## Architecture

The codebase follows a **modular screen pattern**: screen files in `app/(tabs)/` contain only JSX, while all state, effects, and handlers live in dedicated hooks (e.g. `hooks/use-wallet-screen.ts`, `hooks/use-shop-screen.ts`). Sub-components are extracted into `components/wallet/` and `components/shop/` with barrel exports.

Key directories:

- `app/` — Expo Router file-based routing with route groups `(tabs)`, `(auth)`, `(onboarding)`
- `lib/api/` — API client modules (wallet, shop, user, quests, checkout, etc.) with MMKV cache fallback
- `context/` — React Context providers for auth, collectibles, and navbar visibility
- `services/store.ts` — Zustand global store with MMKV persistence
