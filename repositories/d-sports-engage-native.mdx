## Overview

**d-sports-engage-native** (package name: `engage-native`, v1.13.1) is the native mobile app for D-Sports. It mirrors the core PWA experience on iOS and Android: wallet, shop, leaderboard, locker room, and profile.

The app targets both native and web (responsive PWA) and uses the same backend (`d-sports-api`) for API and checkout flows.

## Tech stack

| Category       | Technology                                    |
| -------------- | --------------------------------------------- |
| Framework      | Expo 54, React Native 0.81, React 19          |
| Auth           | Clerk (Expo)                                  |
| Payments       | RevenueCat (react-native-purchases)           |
| Web3           | Thirdweb SDK                                  |
| State          | Zustand + MMKV persistence                    |
| UI             | Lucide React Native                           |
| Navigation     | Expo Router 6 (file-based, typed routes)      |
| Push           | OneSignal                                     |
| Error tracking | Sentry                                        |
| Package        | Bun                                           |

## Features

- **Wallet** — tokens, holdings, pack opening, crypto checkout (Arbitrum, Ethereum, Polygon), external wallet provider integration
- **Shop** — collectibles, cart, coin bundles, fiat checkout (RevenueCat / Apple IAP / Google Play / Stripe) and crypto checkout
- **Leaderboard** — rankings and filters
- **Locker room** — social feed, explore fans and teams, daily games (Pick'em, Spin Wheel, Guess the Player), quests
- **Profile** — user profile editing (avatar/banner upload), followers/following, points history
- **Settings** — API-backed preferences (appearance, notifications, privacy, accessibility, data), blocked users, in-app support tickets, legal pages
- **Onboarding** — new-user onboarding flow with auth protection
- **Push notifications** — OneSignal SDK on iOS, Android, and web with backend subscription sync
- **Theme** — dark/light mode (default dark)

## Project structure

```text
app/
├── (auth)/              # Login, signup, SSO, password reset
├── (onboarding)/        # New user onboarding flow
├── (tabs)/              # Main tab navigation (wallet, shop, leaderboard, locker room, profile)
├── settings/            # Settings pages with nested modals/tabs
└── _layout.tsx          # Root layout with providers + auth protection

components/
├── wallet/              # Extracted wallet sub-components
├── shop/                # Extracted shop sub-components
├── locker-room/         # Locker room components (feed, games, quests, explore)
├── leaderboard/         # Leaderboard components
├── profile/             # Profile stat modals
├── settings/            # Settings items, sections, and modal/tab components
├── ui/                  # Reusable UI primitives (Button, TextField, etc.)
└── Icon/                # Icon wrapper using lucide-react-native

hooks/                   # Screen-level hooks (use-wallet-screen, use-shop-screen, etc.)
lib/api/                 # API client modules (wallet, shop, user, quests, checkout, etc.)
context/                 # React Context providers (user, collectibles, navbar, actions)
services/                # Zustand store, MMKV storage, types
theme/                   # Brand colors, dark/light palettes, design tokens
types/                   # Shared TypeScript types
constants/               # Shared constants (tokens, shop bundles, etc.)
```

## Getting started

<Steps>
  <Step title="Install dependencies">
    ```bash
    bun install
    ```
  </Step>
  <Step title="Configure environment">
    Set the following `EXPO_PUBLIC_*` variables (see `.env` in the repo):

    | Variable | Purpose |
    | -------- | ------- |
    | `EXPO_PUBLIC_CLERK_PUBLISHABLE_KEY` | Clerk authentication |
    | `EXPO_PUBLIC_API_URL` | Backend API base URL |
    | `EXPO_PUBLIC_TW_CLIENT_ID` | Thirdweb Web3 client |
    | `EXPO_PUBLIC_REVENUECAT_API_KEY` | RevenueCat in-app purchases |
    | `EXPO_PUBLIC_SUPABASE_URL` | Supabase client URL |
    | `EXPO_PUBLIC_SUPABASE_KEY` | Supabase publishable key |
  </Step>
  <Step title="Start the development server">
    ```bash
    bunx expo start
    ```
    Press `a` for Android, `i` for iOS, or scan the QR code with Expo Go.
  </Step>
</Steps>

## Build and deploy

The app uses **EAS Build** for native builds and **EAS Update** for OTA updates.

| Command | Description |
| ------- | ----------- |
| `bun run build:dev` | Development build (all platforms) |
| `bun run build:preview` | Internal preview build |
| `bun run build:prod` | Production build |
| `bun run update` | Push an OTA update via EAS Update |
| `bun run submit:ios` | Submit to App Store |

Build profiles are configured in `eas.json` with channels: `development`, `preview`, `staging`, and `production`.

## Architecture patterns

- **Modular screen architecture** — screen files contain only JSX; all state, effects, and handlers live in dedicated hooks (`use-wallet-screen.ts`, `use-shop-screen.ts`)
- **Extracted sub-components** — wallet and shop screens decomposed into `components/wallet/` and `components/shop/` with barrel exports
- **API client layer** — `lib/api/` with MMKV cache fallback for offline resilience
- **Crypto checkout** — Thirdweb SDK calls the PWA backend for on-chain payments (Arbitrum default, Ethereum, Polygon)
- **PWA-ready** — `display: standalone`, responsive desktop layout (`maxWidth: 480px`), web hover states, keyboard nav
