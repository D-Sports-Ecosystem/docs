## Overview

**d-sports-engage-native** (package name: `engage-native`) is the native mobile app for D-Sports. It mirrors the core PWA experience on iOS and Android: wallet, shop, leaderboard, locker room, and profile.

## Getting started

```bash
# Install dependencies
bun install

# Start development server
bunx expo start
# Press 'a' for Android, 'i' for iOS, or scan QR with Expo Go

# TypeScript check
bun tsc --noEmit
```

For development builds, use `bun run build:dev` (EAS) or run with the Expo dev client.

## Features

- **Wallet** — Tokens, holdings, pack opening, crypto checkout (via PWA backend)
- **Shop** — Collectibles, cart, coin bundles, checkout
- **Leaderboard** — Rankings and filters
- **Locker room** — Social feed and engagement
- **Profile** — User profile and settings
- **Theme** — Dark/light mode (default dark)

├── shop/                # Extracted shop sub-components

## Architecture

<Tabs>
  <Tab title="Screen pattern">
    Screen files (`wallet.tsx`, `shop.tsx`) contain **only imports and JSX**. All state, effects, and handlers live in dedicated hooks:
    
    - `hooks/use-wallet-screen.ts` — wallet/token fetch effects, PIN verification, transaction handlers
    - `hooks/use-shop-screen.ts` — cart state, product queries, carousel auto-scroll, checkout logic
    
    Sub-components are extracted into `components/wallet/` and `components/shop/` with barrel exports.
  </Tab>
  <Tab title="State management">
    - **Zustand** store in `services/store.ts` with MMKV persistence
    - **React Context** for auth, collectibles, navbar visibility
    - API client layer in `lib/api/` with MMKV cache fallback
  </Tab>
  <Tab title="Routing">
    - **Expo Router** with file-based routing and typed routes
    - Route groups: `(tabs)`, `(auth)`, `(onboarding)`
    - Auth protection in `_layout.tsx` with automatic redirects
  </Tab>
</Tabs>

## Payments

The app supports both fiat and crypto payments:

- **RevenueCat** (`lib/revenuecat/provider.tsx`) — Apple IAP, Google Play, and Stripe (web)
- **Crypto checkout** (`lib/api/checkout-api.ts`) — calls PWA backend for on-chain payments via Thirdweb SDK
- Supported chains: Arbitrum (default), Ethereum, Polygon
