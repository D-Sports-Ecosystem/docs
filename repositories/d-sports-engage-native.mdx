## Overview

**d-sports-engage-native** (package name: `engage-native`, version `1.16.2`) is the native mobile app for D-Sports. It mirrors the core PWA experience on iOS and Android: wallet, shop, leaderboard, locker room, and profile.

The app also supports **web as a PWA** with `display: standalone`, responsive desktop layout (max-width 480 px), and web hover states.

## Tech stack

| Category    | Technology                                  |
| ----------- | ------------------------------------------- |
| Framework   | Expo 54, React Native 0.81, React 19       |
| Language    | TypeScript 5.9                              |
| Auth        | Clerk (`@clerk/clerk-expo`)                 |
| Payments    | RevenueCat (`react-native-purchases`)       |
| Web3        | Thirdweb SDK (Arbitrum, Ethereum, Polygon)  |
| State       | Zustand 5 + MMKV persistence               |
| Navigation  | Expo Router 6 (file-based, typed routes)    |
| Icons       | Lucide React Native                         |
| Monitoring  | Sentry (`@sentry/react-native`)             |
| Push        | OneSignal (`react-native-onesignal`)        |
| Animations  | React Native Reanimated 4                   |
| Package mgr | Bun 1.3                                     |

## Features

- **Wallet** — Token balances, holdings, pack opening, crypto checkout (Thirdweb → PWA backend)
- **Shop** — Collectibles, cart, coin bundles, fiat (RevenueCat) and crypto checkout
- **Leaderboard** — Global and team-scoped rankings with filters
- **Locker room** — Social feed, quests, daily games (Pick 'Em, Spin Wheel, Guess the Player)
- **Profile** — User profile, achievements, badges, followers/following
- **Settings** — Account, appearance, accessibility, notifications, support, legal, changelog
- **Theme** — Dark/light mode (default dark). Base background `#0a0e1a`, accent gold `#F5C842`, primary blue `#4169E1`

## Environment variables

The app reads `EXPO_PUBLIC_*` environment variables at runtime. You need the following in your `.env`:

| Variable | Purpose |
| -------- | ------- |
| `EXPO_PUBLIC_CLERK_PUBLISHABLE_KEY` | Clerk auth (live publishable key) |
| `EXPO_PUBLIC_API_URL` | Backend API base URL (defaults to `https://api.d-sports.org`) |
| `EXPO_PUBLIC_TW_CLIENT_ID` | Thirdweb client ID for Web3 |
| `EXPO_PUBLIC_REVENUECAT_API_KEY` | RevenueCat API key |
| `EXPO_PUBLIC_REVENUECAT_APPSTORE_ID` | RevenueCat App Store ID |
| `EXPO_PUBLIC_REVENUECAT_ENTITLEMENT` | RevenueCat entitlement identifier |
| `EXPO_PUBLIC_SUPABASE_URL` | Supabase project URL |
| `EXPO_PUBLIC_SUPABASE_KEY` | Supabase publishable key |

## Getting started

<Steps>
  <Step title="Install dependencies">
    ```bash
    bun install
    ```
  </Step>
  <Step title="Configure environment">
    Copy the `.env` template and fill in your keys for Clerk, RevenueCat, Thirdweb, and the API URL.
  </Step>
  <Step title="Start the dev server">
    ```bash
    bunx expo start
    ```
    Press **a** for Android, **i** for iOS, or scan the QR code with Expo Go.
  </Step>
  <Step title="Type-check">
    ```bash
    bun tsc --noEmit
    ```
  </Step>
</Steps>

## EAS build profiles

Native builds and OTA updates use [Expo Application Services (EAS)](https://expo.dev). The project ID is `b8ea114d-c41d-4d5f-87a9-d91901ed588b`.

| Profile | Command | Purpose |
| ------- | ------- | ------- |
| `development` | `bun run build:dev` | Dev client with fast refresh (iOS simulator by default) |
| `development-device` | `eas build --profile development-device --platform ios` | Dev client on a physical iOS device |
| `preview` | `bun run build:preview` | Release-mode APK/IPA for internal QA |
| `production` | `bun run build:prod` | Store-ready build (auto-increments version) |

OTA updates push JS/asset changes without a new store build:

```bash
bun run update --branch production --message "description"
```

## Backend integration surfaces

- **API base URL:** `EXPO_PUBLIC_API_URL` (defaults to `https://api.d-sports.org`).
- The API client in `lib/api/client.ts` injects Clerk bearer tokens automatically, normalizes the `{ success, data?, error?, code? }` response envelope, and retries transient 5xx errors with exponential backoff.
- Domain-specific API modules: `user-api`, `wallet-api`, `shop-api`, `quests-api`, `leaderboard-api`, `locker-room-api`, `teams-api`, `collectibles-api`, `games-api`, `checkout-api`.
- `lib/api/cache.ts` provides MMKV-backed cache-first fetching for read-heavy endpoints.
