## Overview

**d-sports-engage-native** (package name: `engage-native`) is the native mobile app for D-Sports. It mirrors the core PWA experience on iOS and Android: wallet, shop, leaderboard, locker room, and profile. The app also targets the web as a responsive PWA.

## Tech stack

| Category      | Technology                                   |
| ------------- | -------------------------------------------- |
| Framework     | Expo 54, React Native 0.81, React 19         |
| Auth          | Clerk (`@clerk/clerk-expo`)                  |
| Payments      | RevenueCat (`react-native-purchases`)        |
| Web3          | Thirdweb SDK                                 |
| State         | Zustand + MMKV persistence                   |
| UI            | Lucide React Native                          |
| Navigation    | Expo Router 6 (file-based, typed routes)     |
| Push          | OneSignal                                    |
| Error tracking| Sentry (`@sentry/react-native`)              |
| Package manager| Bun 1.3                                     |

## Features

- **Wallet** — tokens, holdings, pack opening, crypto checkout (via PWA backend on Arbitrum, Ethereum, and Polygon)
- **Shop** — collectibles, cart, coin bundles, fiat and crypto checkout
- **Leaderboard** — rankings and filters
- **Locker room** — social feed, quests, daily games (Pick'Em, Spin Wheel, Guess the Player)
- **Teams** — follow and fan teams including HoD (House of David)
- **Profile** — user profile, settings, achievements, and points history
- **Theme** — dark/light mode (default dark)

## Getting started

<Steps>
  <Step title="Install dependencies">
    ```bash
    bun install
    ```
  </Step>
  <Step title="Configure environment variables">
    Create a `.env` file at the project root. Only `EXPO_PUBLIC_*` keys are accessible at runtime.

    | Variable | Purpose |
    | -------- | ------- |
    | `EXPO_PUBLIC_CLERK_PUBLISHABLE_KEY` | Clerk authentication |
    | `EXPO_PUBLIC_API_URL` | Backend API base URL (e.g. `https://api.d-sports.org`) |
    | `EXPO_PUBLIC_TW_CLIENT_ID` | Thirdweb client ID for Web3 |
    | `EXPO_PUBLIC_REVENUECAT_API_KEY` | RevenueCat API key |
    | `EXPO_PUBLIC_REVENUECAT_APPSTORE_ID` | RevenueCat App Store ID |
    | `EXPO_PUBLIC_REVENUECAT_ENTITLEMENT` | RevenueCat entitlement identifier |
    | `EXPO_PUBLIC_SUPABASE_URL` | Supabase project URL |
    | `EXPO_PUBLIC_SUPABASE_KEY` | Supabase publishable key |
  </Step>
  <Step title="Start the development server">
    ```bash
    bunx expo start
    ```
    Press `a` for Android, `i` for iOS, or scan the QR code with Expo Go.
  </Step>
</Steps>

## EAS build profiles

The project uses EAS Build with the following profiles defined in `eas.json`:

| Profile | Channel | Notes |
| ------- | ------- | ----- |
| `development` | `development` | Dev client, internal distribution, iOS simulator |
| `development-device` | `development` | Dev client on physical device |
| `preview` | `preview` | Internal distribution, Android APK |
| `staging` | `staging` | Auto-increment version |
| `production` | `production` | Auto-increment version, App Store / Play Store |

```bash
# Development build (both platforms)
bun run build:dev

# Preview APK (Android)
bun run build:preview:android

# Production build
bun run build:prod
```

OTA updates are published automatically via GitHub Actions on push to `master`.

```text
├── leaderboard/         # Leaderboard table and modal

├── ui/                  # Reusable primitives (Button, TextField, TutorialOverlay)

```

## Architecture highlights

- **Screen / hook separation** — screen files contain only JSX; all state and logic live in dedicated hooks (e.g. `use-wallet-screen.ts`).
- **API client layer** — `lib/api/client.ts` provides an authenticated HTTP client. Domain modules (`wallet-api`, `teams-api`, `quests-api`, etc.) are exposed via `useApi()`.
- **Crypto checkout** — fiat payments go through RevenueCat (Apple IAP, Google Play, Stripe on web). Crypto payments use the Thirdweb SDK to sign on-chain transactions against the PWA backend (`POST /api/checkout/crypto`). Supported chains: Arbitrum (default), Ethereum, Polygon.
- **Path alias** — import with `@/*` which maps to the project root.
- **PWA support** — `display: "standalone"`, responsive `maxWidth: 480px` container on desktop, web hover states.
