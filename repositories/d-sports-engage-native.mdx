## Overview

**d-sports-engage-native** (package name: `engage-native`) is the native mobile app for D-Sports. It mirrors the core PWA experience on iOS and Android: wallet, shop, leaderboard, locker room, and profile.

- **Run:** `bunx expo start` or `bun run start` — then press `a` for Android or `i` for iOS, or scan the QR code with Expo Go.

## Features

- **Wallet** — Tokens, holdings, pack opening, crypto checkout (Arbitrum, Ethereum, Polygon)
- **Shop** — Collectibles, cart, coin bundles, 3D model viewer, crypto checkout
- **Leaderboard** — Rankings and filters
- **Locker room** — Social feed, quests, daily pick'em, spin wheel, guess-the-player games
- **Profile** — User profile, settings, and team membership
- **Onboarding** — New-user onboarding flow with team selection
- **Theme** — Dark/light mode (default dark)

## Getting started

<Steps>
  <Step title="Install dependencies">
    Clone the repository and run `bun install`.
  </Step>
  <Step title="Configure environment">
    Create a `.env` file at the project root with the following `EXPO_PUBLIC_*` variables:

    ```bash
    EXPO_PUBLIC_CLERK_PUBLISHABLE_KEY="your-clerk-key"
    EXPO_PUBLIC_API_URL="https://api.d-sports.org"
    EXPO_PUBLIC_TW_CLIENT_ID="your-thirdweb-client-id"
    EXPO_PUBLIC_REVENUECAT_API_KEY="your-revenuecat-key"
    EXPO_PUBLIC_REVENUECAT_APPSTORE_ID="your-appstore-id"
    EXPO_PUBLIC_REVENUECAT_ENTITLEMENT="premium"
    EXPO_PUBLIC_SUPABASE_URL="your-supabase-url"
    EXPO_PUBLIC_SUPABASE_KEY="your-supabase-anon-key"
    ```

    <Note>Only `EXPO_PUBLIC_*` keys are accessible at runtime in the Expo app.</Note>
  </Step>
  <Step title="Start the dev server">
    Run `bunx expo start`, then press `a` for Android, `i` for iOS, or scan the QR code with Expo Go.
  </Step>
  <Step title="Build with EAS (optional)">
    Use EAS for native builds:

    ```bash
    bun run build:dev          # Development build (all platforms)
    bun run build:preview      # Preview build
    bun run build:prod         # Production build
    bun run build:dev:ios      # iOS only
    bun run build:dev:android  # Android only
    ```
  </Step>
</Steps>

The app targets both native and web (responsive) and uses the same backend (d-sports-api) as the PWA for API and checkout flows.

## Architecture

### Modular screen pattern

Screen files under `app/(tabs)/` contain only JSX. All state, effects, and handlers live in dedicated hooks:

- `hooks/use-wallet-screen.ts` — wallet and token fetch logic, PIN verification, transaction handlers
- `hooks/use-shop-screen.ts` — cart state, product queries, carousel auto-scroll, checkout logic

Sub-components are extracted into `components/wallet/` and `components/shop/` with barrel exports via `index.ts`.

### State management

- **Zustand** (`services/store.ts`) with MMKV persistence for global state (theme, cart, points)
- **React Context** for auth (`context/user-context.tsx`), collectibles, and navbar visibility
- **MMKV** (`services/storage.ts`) for synchronous local storage

### API client layer

The `lib/api/` directory contains domain-specific API modules (wallet, shop, user, quests, leaderboard, locker room, teams, collectibles, checkout). All modules use a shared HTTP client (`lib/api/client.ts`) with automatic auth token injection and an MMKV cache-first fallback (`lib/api/cache.ts`).

### Crypto checkout

Crypto payments use the **Thirdweb SDK** to sign on-chain transactions, calling the PWA backend (`POST /api/checkout/crypto` and `POST /api/checkout/crypto/verify`). Supported chains: Arbitrum (default), Ethereum, and Polygon.

Fiat payments are handled by **RevenueCat** — Apple IAP on iOS, Google Play on Android, and Stripe on web.
