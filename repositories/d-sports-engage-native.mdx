## Overview

**d-sports-engage-native** (package name: `engage-native`, current version **v1.14.0**) is the native mobile app for D-Sports. It mirrors the core PWA experience on iOS and Android: wallet, shop, leaderboard, locker room, and profile.

- **Run:** `bunx expo start` or `bun run start` ‚Äî then press `a` for Android or `i` for iOS, or scan the QR code with Expo Go.

## Features

- **Wallet** ‚Äî Tokens, holdings, pack opening, crypto checkout (via PWA backend)
- **Shop** ‚Äî Collectibles, cart, coin bundles, checkout
- **Leaderboard** ‚Äî Per-team scoring with aggregated global leaderboard, category "See All" modals
- **Locker room** ‚Äî Social feed, daily pick-em, spin wheel, guess-the-player games, and quest sections
- **Quests** ‚Äî Team-scoped (`GLOBAL`, `TEAM`, `EVENT`) questlines with `stepCode`-based progression, favorites gating, and paid-pass (REP/MVP) track support
- **Favorites** ‚Äî Up to 5 favorited teams; quest eligibility and team leaderboard access are tied to favorites
- **Profile** ‚Äî User profile editing with avatar/banner uploads, follow lists, and points history
- **Settings** ‚Äî Accessibility, appearance, notifications, privacy, data, audio, and support pages
- **Theme** ‚Äî Dark/light mode (default dark)

## Environment variables

The app uses `EXPO_PUBLIC_*` keys at runtime. You need to set the following in your `.env` file:

| Variable | Purpose |
| -------- | ------- |
| `EXPO_PUBLIC_CLERK_PUBLISHABLE_KEY` | Clerk authentication |
| `EXPO_PUBLIC_API_URL` | Backend API base URL |
| `EXPO_PUBLIC_TW_CLIENT_ID` | Thirdweb client ID (Web3) |
| `EXPO_PUBLIC_REVENUECAT_API_KEY` | RevenueCat in-app purchases |
| `EXPO_PUBLIC_REVENUECAT_APPSTORE_ID` | RevenueCat App Store identifier |
| `EXPO_PUBLIC_REVENUECAT_ENTITLEMENT` | RevenueCat entitlement name |
| `EXPO_PUBLIC_SUPABASE_URL` | Supabase project URL |
| `EXPO_PUBLIC_SUPABASE_KEY` | Supabase publishable key |

## Getting started

1. Clone the repository and run `bun install`.
2. Copy `.env.example` to `.env` and fill in the values above (Clerk, RevenueCat, Thirdweb, Supabase, API base URL).
3. Run `bunx expo start`.
4. For development builds: `bun run build:dev` (EAS) or run with Expo dev client.

The app targets both native and web (responsive) and uses the same backend (d-sports-api) as the PWA for API and checkout flows.

## v1.14.0 highlights

- **Team-scoped quests** ‚Äî quests are now scoped to `GLOBAL`, `TEAM`, or `EVENT` with per-team progression and `stepCode`-based matching for reliable completion tracking.
- **Favorites gating** ‚Äî team quests unlock only when the team is in your favorites (max 5). Quest payloads include eligibility and lock-reason signals so the UI can explain why a quest is unavailable.
- **Paid-pass support** ‚Äî paid-track team quests are gated by team entitlement checks (initial rollout: Oshawa pass quest gating).
- **Leaderboard aggregation** ‚Äî team quest points accrue per-team; the global D-Sports leaderboard aggregates from team board totals.
- **Team detail modal refresh** ‚Äî redesigned to reflect "favorite to unlock team questline + points" with explicit add/remove controls and disabled-state messaging at max favorites.
- **Category "See All"** ‚Äî leaderboard section "See All" actions now open modal lists and route into team detail flows.
