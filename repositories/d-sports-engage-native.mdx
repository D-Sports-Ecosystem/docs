## Overview

**d-sports-engage-native** is the native mobile app for D-Sports Engage, built with Expo 54 and React Native. It delivers the full fan engagement experience — wallet, shop, leaderboard, locker room, and profile — on iOS, Android, and web (PWA-ready).

The app uses file-based routing via Expo Router and shares the same backend (`d-sports-api`) as the PWA for all API and checkout flows.

## Features

- **Wallet** — Token balances, holdings, pack opening, crypto checkout via Thirdweb
- **Shop** — Collectibles, cart, coin bundles, in-app purchases (RevenueCat)
- **Leaderboard** — Rankings with filters and modals
- **Locker room** — Social feed, quests, daily games (pick-em, spin wheel, guess the player)
- **Profile** — User profile, followers/following, achievements, points history
- **Settings** — Account, appearance, notifications, privacy, accessibility, support
- **Theme** — Dark/light mode (default dark, base `#0a0e1a`)

## Getting started

<Steps>
  <Step title="Clone and install">
    ```bash
    git clone <repo-url>
    cd d-sports-engage-native
    bun install
    ```
  </Step>
  <Step title="Configure environment">
    Create a `.env` file at the project root with the following variables:

    | Variable | Description |
    | -------- | ----------- |
    | `EXPO_PUBLIC_CLERK_PUBLISHABLE_KEY` | Clerk publishable key for authentication |
    | `EXPO_PUBLIC_API_URL` | Backend API base URL (e.g. `https://api.d-sports.org`) |
    | `EXPO_PUBLIC_TW_CLIENT_ID` | Thirdweb client ID for web3 features |
    | `EXPO_PUBLIC_REVENUECAT_API_KEY` | RevenueCat API key for in-app purchases |
    | `EXPO_PUBLIC_REVENUECAT_APPSTORE_ID` | RevenueCat App Store identifier |
    | `EXPO_PUBLIC_REVENUECAT_ENTITLEMENT` | RevenueCat entitlement name (e.g. `premium`) |
    | `EXPO_PUBLIC_SUPABASE_URL` | Supabase project URL |
    | `EXPO_PUBLIC_SUPABASE_KEY` | Supabase publishable (anon) key |

    <Note>Only `EXPO_PUBLIC_*` variables are accessible at runtime in the Expo app.</Note>
  </Step>
  <Step title="Start the dev server">
    ```bash
    bunx expo start
    ```

    Press **a** for Android, **i** for iOS, or scan the QR code with Expo Go.
  </Step>
</Steps>

## Architecture patterns

- **Modular screen architecture** — Screen files contain only JSX. All state, effects, and handlers live in dedicated hooks (e.g. `use-wallet-screen.ts`, `use-shop-screen.ts`).
- **Extracted sub-components** — Complex screens like wallet and shop are decomposed into focused components under `components/wallet/` and `components/shop/` with barrel exports.
- **API client layer** — Centralized in `lib/api/` with MMKV cache fallback (`lib/api/cache.ts`) for offline resilience.
- **Zustand + MMKV** for global state with synchronous persistence. React Context handles auth, collectibles, and navbar visibility.
- **Crypto checkout** — On-chain payments via Thirdweb SDK, calling the PWA backend for transaction verification.
- **PWA-ready** — `display: standalone`, responsive desktop layout, web hover states, and keyboard navigation support.
- **Path alias** — `@/*` maps to the project root for clean imports.

## EAS build profiles

The app uses [EAS Build](https://docs.expo.dev/build/introduction/) for native builds. Profiles are defined in `eas.json`:

| Profile              | Channel       | Description                                    |
| -------------------- | ------------- | ---------------------------------------------- |
| `development`        | development   | Dev client build for simulators                |
| `development-device` | development   | Dev client build for physical devices          |
| `preview`            | preview       | Internal distribution (APK on Android)         |
| `staging`            | staging       | Staging build with auto-increment versioning   |
| `production`         | production    | Production build with auto-increment versioning|

Run a build with:

```bash
eas build --profile <profile-name> --platform <ios|android>
```
