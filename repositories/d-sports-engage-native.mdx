## Overview

**d-sports-engage-native** (package name: `engage-native`, version 1.10.10) is the native mobile app for D-Sports. It mirrors the core PWA experience on iOS and Android: wallet, shop, leaderboard, locker room, and profile.

- **Run:** `bunx expo start` — then press `a` for Android, `i` for iOS, or scan the QR code with Expo Go.
- **URI scheme:** `engagenative://`
- **Bundle ID:** `com.dsports.wallet.beta` (iOS and Android)

## Features

- **Wallet** — Tokens, holdings, pack opening, crypto checkout (via PWA backend on Arbitrum, Ethereum, and Polygon)
- **Shop** — Collectibles, cart, coin bundles, 3D GLB model viewer, crypto checkout
- **Leaderboard** — Rankings and filters
- **Locker room** — Social feed, daily pick'em, daily spin wheel, guess the player, quests, and fan exploration
- **Profile** — User profile, settings, and team membership
- **Onboarding** — New user onboarding flow with team selection
- **Theme** — Dark/light mode (default dark)
- **PWA-ready** — `display: standalone`, responsive desktop layout (max-width 480 px), web hover states

## Architecture patterns

- **Modular screen architecture** — screen files contain only JSX; all state, effects, and handlers live in dedicated hooks (`use-wallet-screen.ts`, `use-shop-screen.ts`)
- **File-based routing** via Expo Router with route groups: `(tabs)`, `(auth)`, `(onboarding)`
- **Zustand + MMKV** for global state with synchronous persistence
- **React Context** for auth, collectibles, and navbar visibility
- **API client layer** in `lib/api/` with MMKV cache fallback (`lib/api/cache.ts`)
- **Crypto checkout** via Thirdweb SDK calling PWA backend for on-chain payments (Arbitrum, Ethereum, Polygon)
- **Path alias** `@/*` maps to the project root

## Environment variables

All runtime variables use the `EXPO_PUBLIC_` prefix. You need to configure:

| Variable | Purpose |
| -------- | ------- |
| `EXPO_PUBLIC_CLERK_PUBLISHABLE_KEY` | Clerk authentication |
| `EXPO_PUBLIC_API_URL` | API backend base URL |
| `EXPO_PUBLIC_TW_CLIENT_ID` | Thirdweb Web3 client ID |
| `EXPO_PUBLIC_REVENUECAT_API_KEY` | RevenueCat in-app purchases |
| `EXPO_PUBLIC_REVENUECAT_APPSTORE_ID` | RevenueCat App Store ID |
| `EXPO_PUBLIC_REVENUECAT_ENTITLEMENT` | RevenueCat entitlement identifier |
| `EXPO_PUBLIC_SUPABASE_URL` | Supabase project URL |
| `EXPO_PUBLIC_SUPABASE_KEY` | Supabase publishable key |

## Getting started

1. Clone the repository and run `bun install`.
2. Copy `.env` and fill in your environment variables (see table above).
3. Run `bunx expo start` to start the development server.
4. Press `a` for Android, `i` for iOS, or scan the QR code with Expo Go.

## Build scripts

| Command | Description |
| ------- | ----------- |
| `bun run start` | Start the Expo development server |
| `bun run android` | Run on Android emulator |
| `bun run ios` | Run on iOS simulator |
| `bun run web` | Start web version |
| `bun run build:dev` | EAS development build (all platforms) |
| `bun run build:preview` | EAS preview build (all platforms) |
| `bun run build:prod` | EAS production build (all platforms) |
| `bun run build:web` | Export static web build |
| `bun run update` | Push an EAS Update (OTA) |

The app uses EAS Build for native builds and EAS Update for over-the-air updates. Platform-specific variants are available (e.g., `build:dev:ios`, `build:dev:android`).

The app targets both native and web (responsive) and uses the same backend (**d-sports-api**) as the PWA for API and checkout flows.
