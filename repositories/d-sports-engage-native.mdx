## Overview

**d-sports-engage-native** (package name: `engage-native`, version **1.14.0**) is the native mobile app for D-Sports. It mirrors the core PWA experience on iOS and Android — wallet, shop, leaderboard, locker room, and profile — and also ships as a responsive web PWA via Expo's web export.

- **Run:** `bunx expo start` — then press `a` for Android, `i` for iOS, or scan the QR code with Expo Go.
- **Bundle IDs:** `com.dsports.wallet.beta` (iOS and Android)
- **Deep-link scheme:** `engagenative://`

## Features

- **Wallet** — Token balances, holdings, pack opening, crypto checkout (via PWA backend), QR codes
- **Shop** — Collectibles, cart, coin bundles, 3D model viewer (Filament on native, Three.js on web), crypto checkout
- **Leaderboard** — Rankings and filters
- **Locker room** — Social feed, quests, daily games (Pick 'Em, Spin Wheel, Guess the Player), explore fans and teams
- **Profile** — User profile, followers/following, points history
- **Settings** — Account, notifications, privacy, accessibility, appearance, support, and help center
- **Onboarding** — New-user onboarding flow with team selection
- **Theme** — Dark/light mode (default dark)

## Environment variables

All runtime variables use the `EXPO_PUBLIC_` prefix. Create a `.env` file at the repository root:

| Variable                              | Purpose                     |
| ------------------------------------- | --------------------------- |
| `EXPO_PUBLIC_CLERK_PUBLISHABLE_KEY`   | Clerk authentication        |
| `EXPO_PUBLIC_API_URL`                 | Backend API base URL        |
| `EXPO_PUBLIC_TW_CLIENT_ID`           | Thirdweb client ID          |
| `EXPO_PUBLIC_REVENUECAT_API_KEY`     | RevenueCat API key          |
| `EXPO_PUBLIC_REVENUECAT_APPSTORE_ID` | RevenueCat App Store ID     |
| `EXPO_PUBLIC_REVENUECAT_ENTITLEMENT` | RevenueCat entitlement name |
| `EXPO_PUBLIC_SUPABASE_URL`           | Supabase project URL        |
| `EXPO_PUBLIC_SUPABASE_KEY`           | Supabase publishable key    |

## Getting started

<Steps>
  <Step title="Install dependencies">
    ```bash
    bun install
    ```
  </Step>
  <Step title="Configure environment">
    Copy `.env.example` (or create `.env`) and fill in the variables listed above. Point `EXPO_PUBLIC_API_URL` at your local or staging d-sports-api instance.
  </Step>
  <Step title="Start the dev server">
    ```bash
    bunx expo start
    ```
    Press `a` for Android, `i` for iOS, or scan the QR code with Expo Go.
  </Step>
  <Step title="Run a development build (optional)">
    ```bash
    bun run build:dev:ios      # iOS simulator
    bun run build:dev:android  # Android emulator
    ```
    These use EAS Build with the `development` profile.
  </Step>
</Steps>

## Architecture patterns

- **File-based routing** via Expo Router with route groups `(tabs)`, `(auth)`, `(onboarding)`.
- **Modular screen architecture** — screen files contain only JSX; all state and handlers live in dedicated hooks (e.g. `use-wallet-screen.ts`, `use-shop-screen.ts`).
- **Extracted sub-components** — wallet and shop screens are decomposed into focused components with barrel exports.
- **Zustand + MMKV** for global state with synchronous persistence.
- **React Context** for auth (`UserContext`), collectibles, and navbar visibility.
- **API client layer** in `lib/api/` with MMKV cache fallback.
- **Crypto checkout** via Thirdweb SDK calling the PWA backend for on-chain payments (Arbitrum, Ethereum, Polygon).
- **PWA-ready** — `display: standalone`, responsive desktop layout (`maxWidth: 480px`), web hover states.
- **Path alias** `@/*` maps to the project root.
