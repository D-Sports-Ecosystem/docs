## Overview

**d-sports-engage-native** (package name: `engage-native`, version `1.10.10`) is the native mobile app for D-Sports. It mirrors the core PWA experience on iOS and Android: wallet, shop, leaderboard, locker room, and profile.

- **Run:** `bunx expo start` — then press `a` for Android, `i` for iOS, or scan the QR code with Expo Go.
- **Bundle IDs:** `com.dsports.wallet.beta` (iOS and Android)
- **Deep link scheme:** `engagenative://`

## Features

- **Wallet** — Token balances, holdings, pack opening, send/receive, crypto checkout via PWA backend
- **Shop** — Collectibles, cart, coin bundles, 3D model viewer (Filament on native, Three.js on web), crypto checkout
- **Leaderboard** — Rankings and filters
- **Locker room** — Social feed, daily pick'em, spin wheel, guess-the-player games, quests, D-Sports Cash
- **Profile** — User profile, settings, accessibility options
- **Theme** — Dark/light mode (default dark)

## Architecture

### Screen pattern

Screen files (`wallet.tsx`, `shop.tsx`) contain **only JSX**. All state, effects, and handlers live in dedicated hooks (`use-wallet-screen.ts`, `use-shop-screen.ts`). Sub-components are extracted into `components/wallet/` and `components/shop/` with barrel exports.

### Routing

Expo Router with file-based routing and typed routes. Route groups:
- `(tabs)` — main tab navigation
- `(auth)` — login, signup, SSO
- `(onboarding)` — new user flow

Auth protection in `_layout.tsx` redirects users based on authentication and onboarding state.

### State management

- **Zustand** store (`services/store.ts`) with synchronous MMKV persistence
- **React Context** for auth (`UserContext`), collectibles, navbar visibility, and action state

### API client

All API calls go through `lib/api/client.ts` with Clerk auth token injection. Domain modules (`wallet-api.ts`, `shop-api.ts`, etc.) are exposed via the `useApi()` hook in `lib/api/index.ts`. An MMKV-based cache layer (`lib/api/cache.ts`) provides offline-first fetching.

### Payments

- **RevenueCat** — fiat payments via Apple IAP (native), Google Play (native), and Stripe (web)
- **Thirdweb SDK** — on-chain crypto payments. The app calls the PWA backend (`POST /api/checkout/crypto`) and signs transactions client-side. Supported chains: Arbitrum (default), Ethereum, Polygon

## Environment variables

All runtime-accessible keys use the `EXPO_PUBLIC_` prefix:

| Variable | Purpose |
| -------- | ------- |
| `EXPO_PUBLIC_CLERK_PUBLISHABLE_KEY` | Clerk authentication |
| `EXPO_PUBLIC_API_URL` | Backend API base URL (`https://api.d-sports.org`) |
| `EXPO_PUBLIC_TW_CLIENT_ID` | Thirdweb client ID |
| `EXPO_PUBLIC_REVENUECAT_API_KEY` | RevenueCat API key |
| `EXPO_PUBLIC_REVENUECAT_APPSTORE_ID` | RevenueCat App Store ID |
| `EXPO_PUBLIC_REVENUECAT_ENTITLEMENT` | RevenueCat entitlement name |
| `EXPO_PUBLIC_SUPABASE_URL` | Supabase project URL |
| `EXPO_PUBLIC_SUPABASE_KEY` | Supabase publishable key |

## Getting started

<Steps>
  <Step title="Install dependencies">
    Clone the repository and run `bun install`.
  </Step>
  <Step title="Configure environment">
    Copy `.env.example` to `.env` and fill in the required keys (Clerk, Thirdweb, RevenueCat, API URL, Supabase).
  </Step>
  <Step title="Start the dev server">
    Run `bunx expo start`, then press `a` for Android, `i` for iOS, or scan the QR code with Expo Go.
  </Step>
  <Step title="Run a development build (optional)">
    Use EAS to create a development build:

    ```bash
    bun run build:dev:ios     # iOS simulator
    bun run build:dev:android # Android
    ```
  </Step>
</Steps>

## EAS build profiles

| Profile       | Description                                | Command |
| ------------- | ------------------------------------------ | ------- |
| `development` | Dev client with simulator support           | `bun run build:dev` |
| `preview`     | Internal distribution (APK on Android)      | `bun run build:preview` |
| `staging`     | Auto-incrementing version, staging channel  | via `eas.json` |
| `production`  | Auto-incrementing version, production channel | `bun run build:prod` |

OTA updates are delivered via `bunx eas-cli update` using the Expo Updates service.

## Web support

The app also runs as a responsive web app (PWA) via `expo start --web`. Key web optimizations:

- `AppScreen` wrapper caps layout at `480px` on desktop
- `WebHoverWrapper` provides hover feedback for Pressables
- PWA manifest configured with `display: "standalone"` and `startUrl: "/"`

The native app and web build share the same backend (`d-sports-api`).
