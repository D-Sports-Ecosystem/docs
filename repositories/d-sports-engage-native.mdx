## Overview

**d-sports-engage-native** (package name: `engage-native`) is the native mobile app for D-Sports. It mirrors the core PWA experience on iOS and Android: wallet, shop, leaderboard, locker room, and profile.

- **Run:** `bunx expo start` or `bun run start` — then press `a` for Android or `i` for iOS, or scan the QR code with Expo Go.

## Tech stack

| Category   | Technology                |
| ---------- | ------------------------- |
| Framework  | Expo 54, React Native 0.81, React 19 |
| Auth       | Clerk (Expo)              |
| Payments   | RevenueCat (react-native-purchases) |
| Web3       | Thirdweb                  |
| State      | Zustand                   |
| Storage    | MMKV                      |
| UI         | Lucide React Native       |
| Navigation | Expo Router               |
| Package    | Bun                       |

## Features

- **Wallet** — Real-time token balances, holdings, pack opening, send/receive, and crypto checkout (via PWA backend)
- **Shop** — Collectibles, cart, coin bundles, checkout
- **Leaderboard** — Rankings and filters
- **Locker room** — Social feed, quests, daily games (Pick'Em, Spin Wheel, Guess the Player)
- **Profile** — User profile and settings
- **Theme** — Dark/light mode (default dark)

### Supported tokens

The wallet fetches live on-chain balances for the following tokens across Ethereum, Polygon, and BSC:

| Token | Type | Chains |
| ----- | ---- | ------ |
| ETH | Native | Ethereum |
| POL | Native | Polygon |
| BNB | Native | BSC |
| USDC | ERC-20 | Ethereum, Polygon, BSC |
| USDT | ERC-20 | Ethereum, Polygon, BSC |
| WBTC | ERC-20 | Ethereum |
| MATIC | ERC-20 | Ethereum |
| DOT | ERC-20 | Ethereum |

<Note>
  DOGE, SOL, and XRP are displayed in the token list but do not yet have on-chain balance fetching (non-EVM).
</Note>

### Wallet providers

You can connect the following external wallets: **MetaMask**, **Coinbase Wallet**, **Trust Wallet**, **Rainbow**, and **Phantom**.

## Getting started

<Steps>
  <Step title="Clone and install">
    Clone the repository and run `bun install`.
  </Step>
  <Step title="Configure environment variables">
    Create a `.env` file with the following keys:

    | Variable | Description |
    | -------- | ----------- |
    | `EXPO_PUBLIC_CLERK_PUBLISHABLE_KEY` | Clerk authentication key |
    | `EXPO_PUBLIC_API_URL` | Backend API base URL (e.g. `https://api.d-sports.org`) |
    | `EXPO_PUBLIC_TW_CLIENT_ID` | Thirdweb client ID for Web3 |
    | `EXPO_PUBLIC_REVENUECAT_API_KEY` | RevenueCat API key for in-app purchases |
    | `EXPO_PUBLIC_REVENUECAT_APPSTORE_ID` | RevenueCat App Store ID |
    | `EXPO_PUBLIC_REVENUECAT_ENTITLEMENT` | RevenueCat entitlement identifier |
    | `EXPO_PUBLIC_SUPABASE_URL` | Supabase project URL |
    | `EXPO_PUBLIC_SUPABASE_KEY` | Supabase publishable key |

    <Warning>
      Only `EXPO_PUBLIC_*` variables are accessible at runtime in the Expo app.
    </Warning>
  </Step>
  <Step title="Start the development server">
    Run `bunx expo start`, then press **a** for Android, **i** for iOS, or scan the QR code with Expo Go.
  </Step>
  <Step title="Build for device (optional)">
    For development builds, run `bun run build:dev` (EAS) or use the Expo dev client.
  </Step>
</Steps>

The app targets both native and web (responsive) and uses the same backend (d-sports-api) as the PWA for API and checkout flows.
