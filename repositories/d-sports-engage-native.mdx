## Overview

**d-sports-engage-native** (package name: `engage-native`) is the native mobile app for D-Sports. It mirrors the core PWA experience on iOS and Android: wallet, shop, leaderboard, locker room, and profile.

- **Run:** `bunx expo start` or `bun run start` — then press `a` for Android or `i` for iOS, or scan the QR code with Expo Go.

## Features

- **Wallet** — Token balances with live pricing, holdings, pack opening, crypto checkout (via PWA backend), and token logo images fetched from Supabase
- **Shop** — Collectibles, cart, coin bundles, checkout
- **Leaderboard** — Rankings and filters
- **Locker room** — Social feed and engagement
- **Profile** — User profile and settings
- **Theme** — Dark/light mode (default dark)

## Architecture

- **File-based routing** via Expo Router with route groups `(tabs)`, `(auth)`, `(onboarding)`.
- **Modular screen architecture** — screen files contain only JSX; all state and handlers live in dedicated hooks (`use-wallet-screen.ts`, `use-shop-screen.ts`).
- **Extracted sub-components** — wallet and shop screens are decomposed into `components/wallet/` and `components/shop/` with barrel exports.
- **Zustand + MMKV** for global state with synchronous persistence.
- **API client layer** in `lib/api/` with MMKV cache fallback (`lib/api/cache.ts`).
- **Crypto checkout** via Thirdweb SDK calling the PWA backend for on-chain payments.

### Wallet token logos

Token logos are SVG files stored in a Supabase `crypto-token-assets` bucket. The `getCryptoTokenLogoUrl()` function in `lib/crypto-token-assets.ts` maps token symbols (ETH, SOL, XRP, etc.) to their Supabase public URLs. Because React Native's built-in `Image` component cannot render remote SVGs, the app uses `expo-image` via a `TokenLogoImage` wrapper component. If a logo fails to load, the row falls back to a text abbreviation of the token symbol.

## Environment variables

You need these `EXPO_PUBLIC_` variables in your `.env` file:

| Variable | Purpose |
| -------- | ------- |
| `EXPO_PUBLIC_API_URL` | Base URL for the d-sports-api backend |
| `EXPO_PUBLIC_CLERK_PUBLISHABLE_KEY` | Clerk authentication |
| `EXPO_PUBLIC_TW_CLIENT_ID` | Thirdweb client ID (on-chain wallet balances) |
| `EXPO_PUBLIC_SUPABASE_URL` | Supabase project URL (token logo storage) |
| `EXPO_PUBLIC_SUPABASE_KEY` | Supabase anon/public key |
| `EXPO_PUBLIC_REVENUECAT_API_KEY` | RevenueCat API key |
| `EXPO_PUBLIC_REVENUECAT_APPSTORE_ID` | RevenueCat App Store product ID |
| `EXPO_PUBLIC_REVENUECAT_ENTITLEMENT` | RevenueCat entitlement identifier |

## Getting started

<Steps>
  <Step title="Install dependencies">
    ```bash
    bun install
    ```
  </Step>
  <Step title="Configure environment">
    Copy `.env.example` to `.env` and fill in the values listed above.
  </Step>
  <Step title="Start the dev server">
    ```bash
    bunx expo start
    ```
    Press `a` for Android, `i` for iOS, or scan the QR code with Expo Go.
  </Step>
  <Step title="Run a development build (optional)">
    ```bash
    bun run build:dev
    ```
    This creates an EAS development build for both platforms.
  </Step>
</Steps>

The app targets both native and web (responsive) and uses the same backend (d-sports-api) as the PWA for API and checkout flows.
