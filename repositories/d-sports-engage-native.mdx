## Overview

**d-sports-engage-native** (package name: `engage-native`, v1.10.10) is the native mobile app for D-Sports. It mirrors the core PWA experience on iOS and Android: wallet, shop, leaderboard, locker room, and profile.

## Tech stack

| Category   | Technology                                      |
| ---------- | ----------------------------------------------- |
| Framework  | Expo 54, React Native 0.81, React 19            |
| Language   | TypeScript 5.9                                   |
| Auth       | Clerk (Expo)                                     |
| Payments   | RevenueCat (Apple IAP, Google Play, Stripe web)  |
| Web3       | Thirdweb (on-chain tx signing)                   |
| State      | Zustand + MMKV persistence                       |
| Navigation | Expo Router (file-based)                         |
| Monitoring | Sentry                                           |
| Database   | Supabase (client-side JS, not direct DB)         |
| Package    | Bun                                              |

## Features

- **Wallet** — token balances, holdings, pack opening, crypto checkout (Arbitrum, Ethereum, Polygon), secure PIN verification, seed phrase management
- **Shop** — collectibles, card packs, coin bundles, cart, crypto and fiat checkout
- **Leaderboard** — team-themed rankings with filters, per-team accent colors
- **Locker room** — social feed, quests, daily games (Pick'Em, Spin Wheel, Guess the Player), D-Sports Cash
- **Profile** — user profile, badges, achievement system, settings
- **Theme** — dark/light mode (default dark)
- **Web** — PWA-ready with responsive desktop layout (`maxWidth: 480px`), hover states, keyboard nav

## EAS builds

The project uses [EAS Build](https://docs.expo.dev/build/introduction/) for native builds and OTA updates. Build profiles are defined in `eas.json`:

| Profile              | Purpose                        | Distribution |
| -------------------- | ------------------------------ | ------------ |
| `development`        | Dev client (iOS simulator)     | Internal     |
| `development-device` | Dev client (physical device)   | Internal     |
| `preview`            | Internal testing (APK on Android) | Internal  |
| `staging`            | Pre-production, auto-increment | Default      |
| `production`         | App Store / Play Store release | Default      |

Common build commands:

```bash
# Development
bun run build:dev          # All platforms
bun run build:dev:ios      # iOS only

# Preview
bun run build:preview      # All platforms

# Production
bun run build:prod         # All platforms
bun run build:prod:ios     # iOS only

# OTA update
bun run update

# Store submission
bun run submit:ios
bun run submit:android
```

## Architecture

### Modular screen pattern

Screen files in `app/(tabs)/` contain only imports and JSX. All state, effects, and handlers live in dedicated hooks:

- `hooks/use-wallet-screen.ts` — wallet/token fetch, PIN verification, transaction handlers
- `hooks/use-shop-screen.ts` — cart state, product queries, carousel auto-scroll, checkout logic

Sub-components are extracted into `components/<domain>/` directories with barrel exports via `index.ts`.

### Routing

[Expo Router](https://docs.expo.dev/router/introduction/) with file-based routing and typed routes:

- `(tabs)` — main tab navigation (wallet, shop, leaderboard, locker room, profile)
- `(auth)` — login, signup, SSO callback, password reset
- `(onboarding)` — new user onboarding flow
- `settings/` — nested settings pages with modals and tabs
- `_layout.tsx` — root layout with provider hierarchy and auth-based redirects

### State management

- **Zustand** store in `services/store.ts` with MMKV persistence
- **React Context** providers in `context/` for auth (`UserContext`), collectibles (`CollectiblesContext`), navbar visibility, and action modal state

### API client layer

The API client in `lib/api/` connects to the d-sports-api backend:

- `client.ts` — base HTTP client with Clerk auth token injection
- `index.ts` — `useApi()` hook exposing all domain modules
- Domain modules: `wallet-api`, `shop-api`, `user-api`, `quests-api`, `leaderboard-api`, `locker-room-api`, `teams-api`, `collectibles-api`, `checkout-api`
- `cache.ts` — `fetchWithCache()` with MMKV cache-first strategy and per-resource TTL

All API responses use the standard `{ success, data }` / `{ success: false, error, code? }` envelope.

### Checkout and payments

- **Fiat** — RevenueCat handles Apple IAP (native), Google Play (native), and Stripe (web) via `lib/revenuecat/provider.tsx`
- **Crypto** — Thirdweb SDK signs on-chain transactions; the checkout flow calls the PWA backend (`POST /api/checkout/crypto` and `POST /api/checkout/crypto/verify`). Supported chains: Arbitrum (default), Ethereum, Polygon.
- **D-Sports Cash** — in-app currency for pack purchases, processed without external payment confirmation

### Path aliases

All imports use the `@/*` alias (maps to project root):

```typescript
import { useUser } from "@/context/user-context";
import { getTokenColors } from "@/lib/token-utils";
import type { Token } from "@/types/wallet.types";
```

## Project structure

```
app/
├── (auth)/              # Login, signup, SSO, password reset
├── (onboarding)/        # New user onboarding flow
├── (tabs)/              # Main tab navigation screens
├── settings/            # Settings pages with modals/tabs
└── _layout.tsx          # Root layout with providers + auth protection

components/
├── wallet/              # Wallet sub-components (TokenRow, ActionModal, etc.)
├── shop/                # Shop sub-components (CartModal, CryptoCheckoutModal, etc.)
├── locker-room/         # Locker room components (feed, games, quests)
├── leaderboard/         # Leaderboard components
├── settings/            # Settings components
├── layout/              # AppScreen responsive wrapper
├── ui/                  # Reusable primitives (Button, TextField, etc.)
└── Icon/                # Icon wrapper (lucide-react-native)

hooks/                   # Screen-level hooks (use-wallet-screen, use-shop-screen, etc.)
types/                   # Shared types (wallet, shop, checkout, API)
constants/               # Shared constants (wallet, shop, locker room)
lib/api/                 # API client modules
context/                 # React Context providers
services/                # Zustand store, MMKV storage, core types
theme/                   # Brand colors, spacing, typography
```
