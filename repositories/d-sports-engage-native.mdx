## Overview

**d-sports-engage-native** (package name: `engage-native`) is the native mobile app for D-Sports. It mirrors the core PWA experience on iOS and Android: wallet, shop, leaderboard, locker room, and profile.

- **Run:** `bunx expo start` or `bun run start` — then press `a` for Android or `i` for iOS, or scan the QR code with Expo Go.

## Tech stack

| Category   | Technology                |
| ---------- | ------------------------- |
| Framework  | Expo 54, React Native 0.81, React 19 |
| Auth       | Clerk (Expo)              |

| Navigation | Expo Router 6             |

| Error tracking | Sentry              |
| Package manager | Bun                |

## Features

- **Wallet** — Token balances, holdings, pack opening, send/receive, and crypto checkout (via PWA backend on Arbitrum, Ethereum, or Polygon)
- **Shop** — Collectibles, cart, coin bundles, 3D model viewer, fiat (RevenueCat) and crypto checkout
- **Leaderboard** — Rankings and filters
- **Locker room** — Social feed, daily pick 'em, spin wheel, guess-the-player games, quests, and team exploration
- **Profile** — User profile, settings, and connected accounts
- **Theme** — Dark/light mode (default dark)

## Environment variables

All runtime variables use the `EXPO_PUBLIC_` prefix. Create a `.env` file in the project root with:

| Variable | Description |
| -------- | ----------- |
| `EXPO_PUBLIC_CLERK_PUBLISHABLE_KEY` | Clerk publishable key for authentication |
| `EXPO_PUBLIC_API_URL` | Backend API base URL (e.g. `https://api.d-sports.org`) |
| `EXPO_PUBLIC_TW_CLIENT_ID` | Thirdweb client ID for Web3 features |
| `EXPO_PUBLIC_REVENUECAT_API_KEY` | RevenueCat API key for in-app purchases |
| `EXPO_PUBLIC_REVENUECAT_APPSTORE_ID` | RevenueCat App Store app ID |
| `EXPO_PUBLIC_REVENUECAT_ENTITLEMENT` | RevenueCat entitlement identifier (e.g. `premium`) |
| `EXPO_PUBLIC_SUPABASE_URL` | Supabase project URL |
| `EXPO_PUBLIC_SUPABASE_KEY` | Supabase publishable (anon) key |

## EAS build profiles

Builds are managed with [EAS Build](https://docs.expo.dev/build/introduction/). Profiles are defined in `eas.json`:

| Profile | Command | Notes |
| ------- | ------- | ----- |
| **development** | `bun run build:dev` | Dev client, internal distribution, iOS simulator enabled |
| **development-device** | — | Same as development but targets physical devices |
| **preview** | `bun run build:preview` | Internal distribution, Android APK output |
| **staging** | — | Auto-increments version, `staging` update channel |
| **production** | `bun run build:prod` | Auto-increments version, `production` update channel |

## API client layer

The app communicates with `d-sports-api` through a typed client in `lib/api/`. Each domain has its own module:

| Module | Covers |
| ------ | ------ |
| `user-api` | Profile, authentication, avatar |
| `wallet-api` | Token balances, transactions, connected wallets |
| `shop-api` | Products, featured packs |
| `checkout-api` | Crypto checkout creation and verification |
| `leaderboard-api` | Rankings, filters |
| `locker-room-api` | Feed posts, comments |
| `quests-api` | Quest listings, progress, claims |
| `teams-api` | Team data, membership |
| `collectibles-api` | Owned packs and items |

Use the `useApi()` hook in any component to access all modules with automatic auth-token injection:

```tsx
const api = useApi();
const result = await api.wallet.getTokens();
```

Responses are cached via MMKV (`lib/api/cache.ts`) so data is available offline.

## Architecture patterns

- **Screen/hook split** — Screen files contain only JSX; all state and logic lives in dedicated hooks (e.g. `use-wallet-screen.ts`).
- **Barrel exports** — Sub-component folders use `index.ts` for clean imports.
- **Path alias** — Import with `@/*` (maps to project root).
- **Crypto checkout** — Thirdweb SDK signs on-chain transactions; the PWA backend handles order creation and verification via `checkout-api`.
- **Fiat payments** — RevenueCat wraps Apple IAP, Google Play, and Stripe (web).
- **PWA-ready** — Responsive desktop layout (`maxWidth: 480px`), web hover states, and `display: standalone` manifest.
