## Overview

**d-sports-engage-native** (package name: `engage-native`) is the native mobile app for D-Sports. It mirrors the core PWA experience on iOS and Android: wallet, shop, leaderboard, locker room, and profile.

- **Run:** `bunx expo start` or `bun run start` ‚Äî then press `a` for Android or `i` for iOS, or scan the QR code with Expo Go.

## Features

- **Wallet** ‚Äî Tokens, holdings, pack opening, crypto checkout (via PWA backend)
- **Shop** ‚Äî Collectibles, cart, coin bundles, checkout
- **Leaderboard** ‚Äî Rankings and filters with team-isolated scoring
- **Locker room** ‚Äî Social feed, quests, rewards, and daily games
- **Quests** ‚Äî Multi-step quest system with scoped questlines (see below)
- **Profile** ‚Äî User profile and settings
- **Theme** ‚Äî Dark/light mode (default dark)

## Quests and rewards

The locker room includes a full quest system powered by the
[gamification quests API](/repositories/d-sports-api/gamification/quests).

### Quest scopes

Every quest carries a **scope** that determines visibility and eligibility:

| Scope | Description |
| -------- | ------------------------------------------------------------ |
| `GLOBAL` | Available to all authenticated users. |
| `TEAM` | Tied to a specific team ‚Äî you must favorite the team first. |
| `EVENT` | Linked to a time-limited event. |

Quests that require a paid pass show `requiresPaidPass: true` and are gated by
team entitlement checks in RevenueCat.

### Filtering

The quest section supports two filter axes:

- **Category** ‚Äî `All`, `Profile`, `Social`, `Purchase`, `Events`
- **Scope** ‚Äî `All`, `GLOBAL`, `TEAM`, `EVENT`, `LOCKED`

Locked quests appear when you haven't met the eligibility requirement (for example,
you haven't favorited the required team). The `lockReason` field explains why.

### Rewards

Rewards are fetched from `GET /api/rewards`. Each reward tracks whether it has been
unlocked, claimed, or redeemed. Superadmins can force-claim or remove rewards
through the admin override endpoints.

### Step progression

Quest steps use a stable `stepCode` identifier for progression matching, falling
back to the legacy title-based match when `stepCode` is not set. This improves
reliability when quest copy changes.

## Favorites

You can favorite up to **5 teams**. Favoriting a team unlocks that team's
questline and contributes points to the team leaderboard. The favorites list
is stored as `favoriteTeamIds` on the user profile.

## Getting started

1. Clone the repository and run `bun install`.
2. Configure environment (Clerk, RevenueCat, Thirdweb, API base URL) per repo README.
3. Run `bunx expo start`.
4. For development builds: `bun run build:dev` (EAS) or run with Expo dev client.

The app targets both native and web (responsive) and uses the same backend (d-sports-api) as the PWA for API and checkout flows.
