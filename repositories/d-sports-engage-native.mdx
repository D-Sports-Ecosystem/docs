## Overview

**d-sports-engage-native** (package name: `engage-native`) is the native mobile app for D-Sports. It mirrors the core PWA experience on iOS and Android: wallet, shop, leaderboard, locker room, and profile.

- **Run:** `bunx expo start` or `bun run start` — then press `a` for Android or `i` for iOS, or scan the QR code with Expo Go.

## Features

- **Wallet** — Tokens, holdings, pack opening, crypto checkout (via PWA backend)
- **Shop** — Collectibles, cart, coin bundles, checkout
- **Leaderboard** — Rankings and filters
- **Locker room** — Social feed, quests, daily games (Pick'Em, Spin Wheel, Guess the Player)
- **Profile** — User profile, followers/following, points history
- **Settings** — Fully featured settings with live API integration (zero mock data)
- **Theme** — Dark/light mode (default dark)

### Settings

The settings page (`app/settings/index.tsx`) is backed by the `useUserSettings` hook (`hooks/use-user-settings.ts`), which calls the d-sports-api for all reads and writes. Settings are grouped into:

| Group | Pages |
| --- | --- |
| Account | Account information, Privacy |
| Preferences | Appearance, Accessibility, Notifications, Data usage |
| Support | Help center, Contact support, Report a problem |
| About | About, Acknowledgements |

Each section is a dedicated route under `app/settings/` with matching tab components in `components/settings/tabs/`. Modals for blocked users, connected accounts, and security live in `components/settings/modals/`.

## Environment variables

The app uses `EXPO_PUBLIC_*` keys that are accessible at runtime:

| Variable | Purpose |
| --- | --- |
| `EXPO_PUBLIC_CLERK_PUBLISHABLE_KEY` | Clerk authentication |
| `EXPO_PUBLIC_API_URL` | Backend API base URL (`https://api.d-sports.org`) |
| `EXPO_PUBLIC_TW_CLIENT_ID` | Thirdweb Web3 client |
| `EXPO_PUBLIC_REVENUECAT_API_KEY` | RevenueCat in-app purchases |
| `EXPO_PUBLIC_REVENUECAT_ENTITLEMENT` | RevenueCat entitlement identifier |
| `EXPO_PUBLIC_SUPABASE_URL` | Supabase client URL |
| `EXPO_PUBLIC_SUPABASE_KEY` | Supabase publishable key |

## Getting started

1. Clone the repository and run `bun install`.
2. Copy `.env.example` to `.env` and fill in the required `EXPO_PUBLIC_*` values listed above.
3. Run `bunx expo start` — press **a** for Android, **i** for iOS, or scan the QR code with Expo Go.
4. For development builds: `bun run build:dev` (EAS) or run with Expo dev client.

The app targets both native and web (responsive) and uses the same backend (d-sports-api) as the PWA for API and checkout flows.
