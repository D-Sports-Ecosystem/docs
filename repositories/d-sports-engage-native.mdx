## Overview

**d-sports-engage-native** (package name: `engage-native`, version 1.14.0) is the native mobile app for D-Sports. It mirrors the core PWA experience on iOS and Android: wallet, shop, leaderboard, locker room, quests, mini-games, and profile.

- **Run:** `bunx expo start` or `bun run start` — then press `a` for Android or `i` for iOS, or scan the QR code with Expo Go.

## Features

- **Wallet** — Tokens, holdings, pack opening, crypto checkout (Arbitrum, Ethereum, Polygon via Thirdweb)
- **Shop** — Collectibles, cart, coin bundles, RevenueCat in-app purchases, crypto checkout
- **Leaderboard** — Global and team rankings with filters
- **Locker room** — Social feed, explore fans and teams, user profile hover cards
- **Quests** — Multi-step quests (global, team, and event scopes) with daily-visit tracking, repeatable quests, cooldowns, and completion verification
- **Mini-games** — Daily spin wheel and daily pick'em
- **Rewards** — Points-based reward system with free and paid tracks (REP and MVP currencies)
- **Profile** — User profile, settings, follow lists, points history
- **Theme** — Dark/light mode (default dark)

## API client layer

All API calls go through the `useApi()` hook exported from `lib/api/index.ts`. Each domain has its own factory function (e.g., `createQuestsApi`, `createGamesApi`) that receives an authenticated config and returns typed methods. Responses are cached locally via MMKV (`lib/api/cache.ts`) for offline fallback.

## Environment variables

Only `EXPO_PUBLIC_*` keys are accessible at runtime. You need to configure:

| Variable                                | Purpose                    |
| --------------------------------------- | -------------------------- |
| `EXPO_PUBLIC_CLERK_PUBLISHABLE_KEY`     | Clerk authentication       |
| `EXPO_PUBLIC_API_URL`                   | d-sports-api backend URL   |
| `EXPO_PUBLIC_TW_CLIENT_ID`             | Thirdweb client ID         |
| `EXPO_PUBLIC_REVENUECAT_API_KEY`       | RevenueCat API key         |
| `EXPO_PUBLIC_REVENUECAT_APPSTORE_ID`   | RevenueCat App Store ID    |
| `EXPO_PUBLIC_REVENUECAT_ENTITLEMENT`   | RevenueCat entitlement     |
| `EXPO_PUBLIC_SUPABASE_URL`             | Supabase project URL       |
| `EXPO_PUBLIC_SUPABASE_KEY`             | Supabase publishable key   |

## Getting started

1. Clone the repository and run `bun install`.
2. Copy `.env` and configure the environment variables listed above.
3. Run `bunx expo start`.
4. For development builds: `bun run build:dev` (EAS) or run with Expo dev client.

## EAS build profiles

The project uses EAS Build with the following profiles defined in `eas.json`:

| Profile                | Description                                    |
| ---------------------- | ---------------------------------------------- |
| `development`          | Dev client, internal distribution, iOS simulator |
| `development-device`   | Dev client on physical devices                 |
| `preview`              | Internal preview builds (APK on Android)       |
| `staging`              | Staging channel with auto-increment            |
| `production`           | Production channel with auto-increment         |

Build commands follow the pattern `bun run build:<profile>` or `bun run build:<profile>:<platform>`.

## Architecture patterns

- **Modular screen architecture** — screen files contain only JSX; all state and handlers live in dedicated hooks (`use-wallet-screen.ts`, `use-shop-screen.ts`)
- **Extracted sub-components** — wallet and shop screens are decomposed into `components/wallet/` and `components/shop/` with barrel exports
- **Zustand + MMKV** for global state with synchronous persistence
- **React Context** for auth (`UserContext`), collectibles, and navbar visibility
- **Crypto checkout** via Thirdweb SDK calling the PWA backend for on-chain payments (Arbitrum default, plus Ethereum and Polygon)
- **PWA-ready** — `display: standalone`, responsive desktop layout, web hover states
- **Path alias** `@/*` maps to project root

The app targets both native and web (responsive) and uses the same backend (d-sports-api) as the PWA for API and checkout flows.
