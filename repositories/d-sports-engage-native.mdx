## Overview

**d-sports-engage-native** (package name: `engage-native`, version 1.13.1) is the native mobile app for D-Sports. It mirrors the core PWA experience on iOS and Android: wallet, shop, leaderboard, locker room, and profile. It also supports web as a responsive PWA.

## Tech stack

| Category    | Technology                                   |
| ----------- | -------------------------------------------- |
| Framework   | Expo 54, React Native 0.81, React 19        |
| Auth        | Clerk (Expo)                                 |
| Payments    | RevenueCat (react-native-purchases)          |
| Web3        | Thirdweb                                     |
| State       | Zustand + MMKV persistence                   |
| UI          | Lucide React Native                          |
| Navigation  | Expo Router 6 (file-based, typed routes)     |
| Animations  | React Native Reanimated 4                    |
| Push        | OneSignal                                    |
| Monitoring  | Sentry                                       |
| Analytics   | Vercel Analytics (web)                        |
| Backend     | Supabase (client-side features)              |
| Package     | Bun                                          |

## Features

- **Wallet** — tokens, holdings, pack opening, crypto checkout (via PWA backend on Arbitrum, Ethereum, or Polygon)
- **Shop** — collectibles, 3D model viewer, cart, coin bundles, crypto and fiat checkout
- **Leaderboard** — rankings and filters
- **Locker room** — social feed, daily pick'em, spin wheel, guess the player, quests
- **Profile** — user profile, followers/following, points history
- **Settings** — account, appearance, notifications, privacy, accessibility, support
- **Onboarding** — new-user onboarding flow with team selection
- **Theme** — dark/light mode (default dark)

## Backend integration surfaces

- Primary API source: `d-sports-api` under `/api/*`.
- Native app consumes team, league, quest, reward, wallet, commerce, and social endpoints.
- Retry/caching logic and auth token injection are centralized in app API client modules.

## Quest and pass UX integration

- Quest UI now reflects team-scoped eligibility and pass-gated progression where applicable.
- Reward presentation differentiates free/unpaid states and claimability.
- Native progression state is expected to mirror backend eligibility and completion semantics.

## Team-aware experiences

- Team discovery, favorite-team interactions, and leaderboard context are integrated across locker room and profile surfaces.
- Team and league metadata is sourced from backend sync pipelines fed by the `leagues` repository.

## Compliance and legal: pack opening odds

- Native odds disclosure behavior is documented with policy-aligned copy and evidence requirements.
- Disclosure values must come from backend payloads and remain visible/accessible near open actions.

## Getting started

### Prerequisites

- [Bun](https://bun.sh) 1.3.9+
- [Expo CLI](https://docs.expo.dev/get-started/set-up-your-environment/)
- For native builds: Xcode (iOS) or Android Studio (Android)
- An [EAS](https://expo.dev/eas) account for cloud builds

### Environment variables

Create a `.env` file at the project root. Only `EXPO_PUBLIC_*` keys are accessible at runtime.

| Variable | Description |
| -------- | ----------- |
| `EXPO_PUBLIC_CLERK_PUBLISHABLE_KEY` | Clerk authentication publishable key |
| `EXPO_PUBLIC_API_URL` | Backend API base URL (e.g. `https://api.d-sports.org`) |
| `EXPO_PUBLIC_TW_CLIENT_ID` | Thirdweb client ID for web3 features |
| `EXPO_PUBLIC_REVENUECAT_API_KEY` | RevenueCat API key for in-app purchases |
| `EXPO_PUBLIC_REVENUECAT_APPSTORE_ID` | RevenueCat App Store ID |
| `EXPO_PUBLIC_REVENUECAT_ENTITLEMENT` | RevenueCat entitlement identifier (e.g. `premium`) |
| `EXPO_PUBLIC_SUPABASE_URL` | Supabase project URL |
| `EXPO_PUBLIC_SUPABASE_KEY` | Supabase publishable (anon) key |

### Install and run

```bash
bun install
bunx expo start
```

Press **a** for Android, **i** for iOS, or scan the QR code with Expo Go.

### Build commands

```bash
# Development builds (EAS)
bun run build:dev            # all platforms
bun run build:dev:ios        # iOS simulator
bun run build:dev:android    # Android

# Preview builds (internal distribution)
bun run build:preview        # all platforms
bun run build:preview:ios
bun run build:preview:android

# Production builds
bun run build:prod           # all platforms
bun run build:prod:ios
bun run build:prod:android

# OTA updates
bun run update

# Store submission
bun run submit:ios
bun run submit:android

# Web export
bun run build:web

# TypeScript check
bun tsc --noEmit
```

## Architecture

### Modular screen pattern

Screen files contain only imports and JSX. All state, effects, and handlers live in dedicated hooks:

- `hooks/use-wallet-screen.ts` — wallet/token fetching, PIN verification, transaction handlers
- `hooks/use-shop-screen.ts` — cart state, product queries, carousel auto-scroll, checkout logic

Sub-components are extracted into `components/wallet/` and `components/shop/` with barrel exports via `index.ts`.

### Routing

[Expo Router](https://docs.expo.dev/router/introduction/) with file-based routing and typed routes. Route groups:

- `(tabs)` — main tab navigation with a custom animated pill tab bar
- `(auth)` — login, signup, SSO callback, password reset
- `(onboarding)` — new user onboarding

Auth protection in `_layout.tsx` automatically redirects based on authentication and onboarding state.

### API client

The `useApi()` hook (in `lib/api/index.ts`) provides access to all API modules with automatic auth token injection:

```typescript
import { useApi } from "@/lib/api";

const api = useApi();
const result = await api.user.getProfile();
```

Available modules: `user`, `quests`, `leaderboard`, `wallet`, `lockerRoom`, `teams`, `collectibles`, `games`, `shop`, `checkout`.

All API calls go to the backend defined by `EXPO_PUBLIC_API_URL`.

### Payments

- **Fiat** — RevenueCat handles Apple IAP (native), Google Play (native), and Stripe (web) via `lib/revenuecat/provider.tsx`
- **Crypto** — Thirdweb SDK for on-chain transaction signing, calling the PWA backend (`POST /api/checkout/crypto` and `POST /api/checkout/crypto/verify`). Supported chains: Arbitrum (default), Ethereum, Polygon.

### State management

- **Zustand** store in `services/store.ts` with MMKV synchronous persistence
- **React Context** providers for authentication (`UserContext`), collectibles, navbar visibility, and action modal state

### Path alias

Import using the `@/*` alias, which maps to the project root:

```typescript
import { useUser } from "@/context/user-context";
import type { Token } from "@/types/wallet.types";
```
