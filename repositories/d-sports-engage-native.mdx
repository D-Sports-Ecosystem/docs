## Overview

**d-sports-engage-native** (package name: `engage-native`, version `1.10.10`) is the native mobile app for D-Sports. It mirrors the core PWA experience on iOS and Android: wallet, shop, leaderboard, locker room, and profile. The app also runs as a responsive web app with PWA support.

## Tech stack

| Category   | Technology                |
| ---------- | ------------------------- |
| Framework  | Expo 54, React Native 0.81, React 19 |
| Auth       | Clerk (Expo)              |
| Payments   | RevenueCat (react-native-purchases) |
| Web3       | Thirdweb                  |
| State      | Zustand 5                 |
| Storage    | MMKV 4                    |
| Animation  | Reanimated 4              |
| UI         | Lucide React Native       |
| Navigation | Expo Router 6             |
| Package    | Bun                       |

## Features

- **Wallet** — tokens, holdings, pack opening, crypto checkout (via PWA backend)
- **Shop** — collectibles, cart, coin bundles, checkout with hidden-product filtering for retired items
- **Leaderboard** — rankings and filters
- **Locker room** — social feed, quests, daily games (Pick 'Em, Spin Wheel, Guess the Player)
- **Profile** — user profile, settings, and team management
- **Theme** — dark/light mode (default dark, base `#0a0e1a`)

## Architecture patterns

- **File-based routing** via Expo Router with route groups `(tabs)`, `(auth)`, `(onboarding)`
- **Modular screen architecture** — screen files contain only JSX; all state, effects, and handlers live in dedicated hooks (`use-wallet-screen.ts`, `use-shop-screen.ts`)
- **Extracted sub-components** — wallet and shop screens decomposed into `components/wallet/` and `components/shop/` with barrel exports
- **API client layer** in `lib/api/` with MMKV cache fallback (`lib/api/cache.ts`)
- **Zustand + MMKV** for global state with synchronous persistence
- **React Context** for auth, collectibles, navbar visibility, and accessibility
- **Crypto checkout** via Thirdweb SDK calling the PWA backend for on-chain payments
- **Path alias** `@/*` maps to the project root

## Environment variables

All runtime variables use the `EXPO_PUBLIC_` prefix. Create a `.env` file at the repo root:

| Variable | Purpose |
| -------- | ------- |
| `EXPO_PUBLIC_CLERK_PUBLISHABLE_KEY` | Clerk authentication (live instance) |
| `EXPO_PUBLIC_API_URL` | Backend API base URL (`https://api.d-sports.org`) |
| `EXPO_PUBLIC_TW_CLIENT_ID` | Thirdweb client ID for Web3 features |
| `EXPO_PUBLIC_REVENUECAT_API_KEY` | RevenueCat API key for in-app purchases |
| `EXPO_PUBLIC_REVENUECAT_APPSTORE_ID` | RevenueCat App Store ID |
| `EXPO_PUBLIC_REVENUECAT_ENTITLEMENT` | RevenueCat entitlement identifier (e.g. `premium`) |
| `EXPO_PUBLIC_SUPABASE_URL` | Supabase project URL (client-side features only) |
| `EXPO_PUBLIC_SUPABASE_KEY` | Supabase publishable key |

## Getting started

<Steps>
  <Step title="Install dependencies">
    ```bash
    bun install
    ```
  </Step>
  <Step title="Configure environment">
    Copy the env template and fill in the values listed above:
    ```bash
    cp .env.example .env
    ```
  </Step>
  <Step title="Start the dev server">
    ```bash
    bunx expo start
    ```
    Press **a** for Android, **i** for iOS, or scan the QR code with Expo Go.
  </Step>
</Steps>

## Build and deploy

The project uses **EAS Build** for native builds. Common scripts:

| Command | Description |
| ------- | ----------- |
| `bun run build:dev` | Development build (all platforms) |
| `bun run build:preview` | Preview/TestFlight build (all platforms) |
| `bun run build:prod` | Production build (all platforms) |
| `bun run build:dev:ios` | Development build (iOS only) |
| `bun run build:dev:android` | Development build (Android only) |
| `bun run update` | Push an OTA update via EAS Update |
| `bun run submit` | Submit to App Store and Play Store |

For web builds, run `bun run build:web` to export a static bundle.
