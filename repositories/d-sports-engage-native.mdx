## Overview

**d-sports-engage-native** (package name: `engage-native`, current version: **1.17.2**) is the native mobile app for D-Sports. It mirrors the core PWA experience on iOS and Android: wallet, shop, leaderboard, locker room, and profile.

- **Run:** `bunx expo start` ‚Äî then press `a` for Android or `i` for iOS, or scan the QR code with Expo Go.

## Tech stack

| Category   | Technology                |
| ---------- | ------------------------- |
| Framework  | Expo 54, React Native 0.81, React 19 |
| Auth       | Clerk (Expo)              |
| Payments   | RevenueCat (react-native-purchases) |
| Web3       | Thirdweb                  |
| State      | Zustand + MMKV            |
| Push       | OneSignal                 |
| Monitoring | Sentry                    |
| Navigation | Expo Router 6 (file-based, typed routes) |
| Package    | Bun                       |

## Features

- **Wallet** ‚Äî Token balances, holdings, pack opening, crypto checkout (via PWA backend), send/receive
- **Shop** ‚Äî Collectibles, cart, coin bundles, 3D model viewer, crypto and fiat checkout
- **Leaderboard** ‚Äî Per-team and global rankings with stack routes (`/leaderboard/[teamSlug]`)
- **Locker room** ‚Äî Social feed, quest center, daily spin wheel, guess-the-player, explore teams/fans
- **Profile** ‚Äî Milestone avatar rings, achievement badges, points history, follow/followers
- **Settings** ‚Äî Account, appearance, accessibility, notifications, support, changelog, legal
- **Theme** ‚Äî Dark/light mode (default dark)

## Environment variables

The app uses `EXPO_PUBLIC_*` keys at runtime. Create a `.env` file at the repo root:

| Variable | Purpose |
| -------- | ------- |
| `EXPO_PUBLIC_CLERK_PUBLISHABLE_KEY` | Clerk authentication |
| `EXPO_PUBLIC_API_URL` | Backend API base URL (e.g. `https://api.d-sports.org`) |
| `EXPO_PUBLIC_TW_CLIENT_ID` | Thirdweb client ID for Web3 |
| `EXPO_PUBLIC_REVENUECAT_API_KEY` | RevenueCat API key |
| `EXPO_PUBLIC_REVENUECAT_APPSTORE_ID` | RevenueCat App Store identifier |
| `EXPO_PUBLIC_REVENUECAT_ENTITLEMENT` | RevenueCat entitlement name (e.g. `premium`) |
| `EXPO_PUBLIC_SUPABASE_URL` | Supabase project URL |
| `EXPO_PUBLIC_SUPABASE_KEY` | Supabase publishable key |

## Getting started

<Steps>
  <Step title="Install dependencies">
    ```bash
    bun install
    ```
  </Step>
  <Step title="Configure environment">
    Copy `.env.example` to `.env` and fill in your Clerk, RevenueCat, Thirdweb, Supabase, and API keys.
  </Step>
  <Step title="Start the dev server">
    ```bash
    bunx expo start
    ```
    Press `a` for Android, `i` for iOS, or scan the QR code with Expo Go.
  </Step>
  <Step title="Run a development build (optional)">
    ```bash
    bun run build:dev:ios      # iOS dev build via EAS
    bun run build:dev:android  # Android dev build via EAS
    ```
  </Step>
</Steps>

## EAS build profiles

| Profile | Channel | Notes |
| ------- | ------- | ----- |
| `development` | `development` | Dev client, internal distribution, iOS simulator |
| `development-device` | `development` | Dev client on physical device |
| `preview` | `preview` | Internal APK for Android |
| `staging` | `staging` | Auto-increment version |
| `production` | `production` | Auto-increment, App Store / Play Store |

Run builds with `bun run build:<profile>` or `bun run build:<profile>:<platform>`.

## Backend integration surfaces

- Primary API source: `d-sports-api` under `/api/*`.
- The native app consumes team, league, quest, reward, wallet, commerce, social, and leaderboard endpoints.
- Retry/caching logic and auth token injection are centralized in `lib/api/client.ts`.
- MMKV-backed cache fallback in `lib/api/cache.ts`.

## Recent changes (v1.17.x)

- **Milestone avatar rings** ‚Äî Profile photo uses a tier-colored SVG ring based on reward milestones (Starter through Legend). Users with fewer than two claimed rewards keep the brand gold ring.
- **Per-team leaderboard routes** ‚Äî `/leaderboard/[teamSlug]` and `/leaderboard/global` stack routes; team boards load by team ID.
- **Points history** ‚Äî Profile modal loads `GET /api/user/points-history` so game awards appear alongside quests.
- **Locker room games as routes** ‚Äî Prize Wheel and Guess the Player open as dedicated stack routes instead of nested modals.
- **Clerk email/password auth** ‚Äî Login supports Clerk Core 3 email + password with 2FA (authenticator, SMS, backup code).
- **Achievement detail modal** ‚Äî Tapping an achievement opens a modal with badge image, title, description, and rarity styling.
