## Overview

**d-sports-engage-native** (package name: `engage-native`) is the native mobile app for D-Sports. It mirrors the core PWA experience on iOS and Android: wallet, shop, leaderboard, locker room, and profile. The app also supports web via Expo's metro bundler with a responsive desktop layout.

## Tech stack

| Category       | Technology                                    |
| -------------- | --------------------------------------------- |
| Framework      | Expo 54, React Native 0.81, React 19          |
| Auth           | Clerk (Expo)                                  |
| Payments       | RevenueCat (react-native-purchases)           |
| Web3           | Thirdweb                                      |
| State          | Zustand                                       |
| Storage        | MMKV                                          |
| UI             | Lucide React Native                           |
| Navigation     | Expo Router 6 (file-based routing)            |
| Monitoring     | Sentry                                        |
| Notifications  | OneSignal                                     |
| Package        | Bun                                           |

## Features

- **Wallet** — tokens, holdings, pack opening, crypto checkout (Thirdweb SDK calling the PWA backend for on-chain payments on Arbitrum, Ethereum, and Polygon)
- **Shop** — collectibles, cart, coin bundles, fiat checkout (RevenueCat) and crypto checkout
- **Leaderboard** — rankings and filters
- **Locker room** — social feed, quests, daily games (Pick 'Em, Spin Wheel, Guess the Player)
- **Profile** — user profile, achievements, followers/following, points history
- **Settings** — account, appearance, accessibility, notifications, privacy, data, support, and changelog
- **Theme** — dark/light mode (default dark)

## Getting started

<Steps>
  <Step title="Install dependencies">
    ```bash
    bun install
    ```
  </Step>
  <Step title="Configure environment variables">
    Create a `.env` file at the project root. Only `EXPO_PUBLIC_*` keys are accessible at runtime:

    | Variable | Description |
    | -------- | ----------- |
    | `EXPO_PUBLIC_CLERK_PUBLISHABLE_KEY` | Clerk authentication publishable key |
    | `EXPO_PUBLIC_API_URL` | Backend API base URL (e.g. `https://api.d-sports.org`) |
    | `EXPO_PUBLIC_TW_CLIENT_ID` | Thirdweb client ID for Web3 features |
    | `EXPO_PUBLIC_REVENUECAT_API_KEY` | RevenueCat API key for in-app purchases |
    | `EXPO_PUBLIC_REVENUECAT_APPSTORE_ID` | RevenueCat App Store app ID |
    | `EXPO_PUBLIC_REVENUECAT_ENTITLEMENT` | RevenueCat entitlement identifier |
    | `EXPO_PUBLIC_SUPABASE_URL` | Supabase project URL (client-side features only) |
    | `EXPO_PUBLIC_SUPABASE_KEY` | Supabase publishable key |
  </Step>
  <Step title="Start the development server">
    ```bash
    bunx expo start
    ```

    Press `a` for Android, `i` for iOS, or scan the QR code with Expo Go.
  </Step>
</Steps>

## Build commands

```bash
# Development builds (EAS)
bun run build:dev          # All platforms
bun run build:dev:ios      # iOS only
bun run build:dev:android  # Android only

# Preview builds (internal distribution)
bun run build:preview          # All platforms
bun run build:preview:ios      # iOS only
bun run build:preview:android  # Android only (APK)

# Production builds
bun run build:prod          # All platforms
bun run build:prod:ios      # iOS only
bun run build:prod:android  # Android only

# OTA updates
bun run update

# Web export
bun run build:web

# TypeScript check
bun tsc --noEmit
```

## Architecture

- **Modular screens** — screen files contain only JSX; all state, effects, and handlers live in dedicated hooks (`use-wallet-screen.ts`, `use-shop-screen.ts`)
- **Extracted sub-components** — wallet and shop screens are decomposed into focused components under `components/wallet/` and `components/shop/` with barrel exports
- **File-based routing** — Expo Router with route groups `(tabs)`, `(auth)`, and `(onboarding)`, plus typed routes
- **Zustand + MMKV** — global state with synchronous persistence for theme, cart, and points
- **API client layer** — `lib/api/client.ts` provides a base HTTP client with auth token injection and MMKV cache fallback
- **Crypto checkout** — Thirdweb SDK signs on-chain transactions via the PWA backend (`POST /api/checkout/crypto`)
- **RevenueCat** — handles fiat payments through Apple IAP, Google Play, and Stripe (web)
- **PWA-ready** — `display: standalone`, responsive desktop layout, web hover states, and keyboard navigation

## CI/CD

The repository uses **EAS Build** for native builds and **EAS Update** for OTA updates. A GitHub Actions workflow (`.github/workflows/eas-update.yml`) automatically publishes OTA updates to iOS when changes are pushed to the `main` or `develop` branches.
