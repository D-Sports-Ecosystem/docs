## Overview

**d-sports-engage-native** (package name: `engage-native`) is the native mobile app for D-Sports. It mirrors the core PWA experience on iOS and Android: wallet, shop, leaderboard, locker room, games, and profile.

- **Run:** `bunx expo start` or `bun run start` — then press `a` for Android or `i` for iOS, or scan the QR code with Expo Go.

## Tech stack

| Category       | Technology                              |
| -------------- | --------------------------------------- |
| Framework      | Expo 54, React Native 0.81, React 19   |
| Auth           | Clerk (Expo)                            |
| Payments       | RevenueCat (react-native-purchases)     |
| Web3           | Thirdweb                                |
| State          | Zustand                                 |
| Storage        | MMKV                                    |
| UI             | Lucide React Native                     |
| Navigation     | Expo Router                             |
| Notifications  | OneSignal                               |
| Error tracking | Sentry                                  |
| Database       | Supabase (client-side)                  |
| Package        | Bun                                     |

## Features

- **Wallet** — Tokens, holdings, pack opening, crypto checkout (via PWA backend)
- **Shop** — Collectibles, cart, coin bundles, checkout
- **Leaderboard** — Rankings and filters
- **Locker room** — Social feed, team exploration, and fan engagement
- **Games** — Daily spin wheel, daily pick-em, and guess-the-player mini-games within the locker room
- **Quests** — Multi-step quest system with daily visit tracking and point rewards
- **Profile** — User profile, achievements, followers, and points history
- **Notifications** — Push notifications via OneSignal
- **Theme** — Dark/light mode (default dark)

## Getting started

<Steps>
  <Step title="Install dependencies">
    Clone the repository and run `bun install`.
  </Step>
  <Step title="Configure environment">
    Create a `.env` file with the following keys (all prefixed with `EXPO_PUBLIC_`):

    | Variable | Description |
    | -------- | ----------- |
    | `EXPO_PUBLIC_CLERK_PUBLISHABLE_KEY` | Clerk authentication publishable key |
    | `EXPO_PUBLIC_API_URL` | API backend URL (e.g. `https://api.d-sports.org`) |
    | `EXPO_PUBLIC_TW_CLIENT_ID` | Thirdweb client ID for Web3 features |
    | `EXPO_PUBLIC_REVENUECAT_API_KEY` | RevenueCat API key for in-app purchases |
    | `EXPO_PUBLIC_REVENUECAT_APPSTORE_ID` | RevenueCat App Store ID |
    | `EXPO_PUBLIC_REVENUECAT_ENTITLEMENT` | RevenueCat entitlement identifier |
    | `EXPO_PUBLIC_SUPABASE_URL` | Supabase project URL |
    | `EXPO_PUBLIC_SUPABASE_KEY` | Supabase publishable key |
  </Step>
  <Step title="Start the dev server">
    Run `bunx expo start`, then press `a` for Android, `i` for iOS, or scan the QR code with Expo Go.
  </Step>
  <Step title="Build for devices (optional)">
    Use EAS for native builds:

    ```bash
    bun run build:dev          # Development build (all platforms)
    bun run build:preview      # Preview/internal build
    bun run build:prod         # Production build
    ```
  </Step>
</Steps>

The app targets both native and web (responsive) and uses the same backend (d-sports-api) as the PWA for API and checkout flows.

## API client

The app uses a modular API client layer in `lib/api/`. You access all API modules through the `useApi()` hook:

```typescript
import { useApi } from "@/lib/api";

const api = useApi();
const profile = await api.user.getProfile();
const quests = await api.quests.getQuests();
const spin = await api.games.awardDailySpinReward(payload);
```

Available modules: `user`, `wallet`, `shop`, `checkout`, `collectibles`, `games`, `quests`, `leaderboard`, `lockerRoom`, and `teams`.
