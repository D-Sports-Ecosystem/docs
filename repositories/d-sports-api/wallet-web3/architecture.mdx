## High-level flow

```mermaid
flowchart LR
  walletRequest[WalletRequest] --> walletProvision[ProvisionOrLookupWallet]
  walletProvision --> keySecurity[PinAndEncryptionFlow]
  keySecurity --> privateKeyGate[PrivateKeyAccessGate]
  walletProvision --> providerBridge[Web3ProviderBridge]
  providerBridge --> balancePriceRead[BalanceAndPriceReads]
```

## Architectural notes

- Wallet secrets are protected via encryption and PIN-gated flows.
- Web3 provider proxy routes isolate external provider contracts from clients.
- Checkout and token balance features consume wallet/web3 utilities as dependencies.
