## Board types

- **Global board** — the single `isGlobal: true` leaderboard. Its scores are derived entirely from `PointsHistory` (see [global scoring](#global-scoring-from-points-history) below).
- **Team boards** — non-global boards tied to a team via `Leaderboard.teamId`. Created on demand by `getOrCreateTeamLeaderboard()`.

## Resolution behavior

`GET /api/leaderboard` supports:

- `?teamId=<teamId>` — resolves (or creates) the team board. Preferred by clients over `leaderboardId`.
- `?leaderboardId=<id>` — direct board lookup by numeric ID.
- No params — returns the global board (`isGlobal: true`).

On every request the endpoint upserts the authenticated user into the resolved board so they always appear in results.

## Global scoring from points history

`PointsHistory` is the **single source of truth** for global leaderboard scores. The function `applySyncUserGlobalAggregateFromTeamBoards()` in `lib/leaderboard.ts` recomputes the global entry for a user by summing all of their `PointsHistory` rows (regardless of `leaderboardId` or `seasonId`) and upserting the total into the global `LeaderboardEntry`.

This means:

- The global board always reflects the user's lifetime earned points.
- Team boards maintain their own `LeaderboardEntry.points` independently.
- Server actions, cron jobs, and backfill scripts call `syncUserGlobalAggregateFromTeamBoards()` to keep the global board in sync after point-awarding events.

## Team board scoring

Team boards aggregate `PointsHistory` by:

- Matching active `seasonId`
- Matching the requested board's `leaderboardId`

Points writes should include board and season scope when context is available.

## Enrollment and write behavior

- `ensureUserIsEnrolledInOGLeaderboard()` creates a zero-point entry for a user on any board if one does not exist.
- `getUserLeaderboards()` currently returns only the global board ID.
- Quest completion, daily visits, and onboarding flows write to `PointsHistory` and then sync the global aggregate.

## Notes

- Stream, search, and stats leaderboard routes are available for read-only consumption.
- Reset and archive lifecycle routes exist but are not covered in this non-admin gamification deep dive.
