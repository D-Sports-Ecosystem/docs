## What this page tracks

This page tracks documentation completeness for **all non-admin backend feature domains** in `d-sports-api`.

Status values used in this page:

- `Documented`
- `Partially documented`
- `Not documented yet`

## Domain coverage matrix (non-admin backend)

- **Gamification** — `Documented`  
  Quests, points, leaderboards, rewards, and current achievements behavior are covered in dedicated subsections.

- **Auth, user profile, onboarding** — `Partially documented`  
  High-level mention exists, but no deep-dive behavior docs yet for auth/session/profile/onboarding lifecycle.

- **Social / locker room** — `Not documented yet`  
  No dedicated behavior or architecture page yet for posts, comments, reactions, and follow graph.

- **Commerce / purchases / catalog** — `Not documented yet`  
  No dedicated non-admin backend behavior docs yet for product, purchase, and payment lifecycle.

- **Collectibles / packs / inventory / minting** — `Not documented yet`  
  Feature is listed at repository level only; no detailed backend flow docs yet.

- **Wallet / web3 integrations** — `Not documented yet`  
  Feature is listed at repository level only; no dedicated backend behavior docs yet.

- **Moderation / reporting** — `Not documented yet`  
  No dedicated documentation section yet for report lifecycle and moderation flows.

- **Platform/infrastructure (non-admin operational endpoints/scripts)** — `Partially documented`  
  Some operational behavior is implied in API/reference and repository notes, but no dedicated operational runbook pages.

## Known exclusions

- Admin and superadmin-only features, controls, and overrides.
- Privileged operational paths intended only for staff-level workflows.
