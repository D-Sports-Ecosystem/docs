## What this page tracks

This page tracks documentation completeness for **all non-admin backend feature domains** in `d-sports-api`.

Status values used in this page:

- `Documented`
- `Partially documented`
- `Not documented yet`

## Domain coverage matrix (non-admin backend)

- **Gamification** — `Documented`  
  Quests, points, leaderboards, rewards, and achievements are documented with dedicated pages.

- **Auth, user profile, onboarding** — `Documented`  
  Dedicated overview, architecture, and behavior docs exist under `/repositories/d-sports-api/auth-onboarding/*`.

- **Social / locker room** — `Documented`  
  Dedicated overview, architecture, and behavior docs exist under `/repositories/d-sports-api/social-locker-room/*`.

- **Commerce / purchases / catalog** — `Documented`  
  Dedicated overview, architecture, and behavior docs exist under `/repositories/d-sports-api/commerce/*`.

- **Collectibles / packs / inventory / minting** — `Documented`  
  Dedicated overview, architecture, and behavior docs exist under `/repositories/d-sports-api/collectibles/*`.

- **Wallet / web3 integrations** — `Documented`  
  Dedicated overview, architecture, and behavior docs exist under `/repositories/d-sports-api/wallet-web3/*`.

- **Moderation / reporting** — `Documented`  
  Dedicated overview, architecture, and behavior docs exist under `/repositories/d-sports-api/moderation/*`.

- **Platform/infrastructure (non-admin operational endpoints/scripts)** — `Partially documented`  
  Dedicated pages exist under `/repositories/d-sports-api/platform-infra/*`, but runbook-level operating procedures remain intentionally lightweight.

## Known exclusions

- Admin and superadmin-only features, controls, and overrides.
- Privileged operational paths intended only for staff-level workflows.
