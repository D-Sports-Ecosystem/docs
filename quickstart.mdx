## Get started in three steps

Run the D-Sports Documentation site locally, or jump to running one of the apps (PWA, site, native, Mic'd Up).

### Step 1: Run the docs locally

<AccordionGroup>
  <Accordion icon="copy" title="Clone the docs repo">
    Clone the D-Sports docs repository and open it in your editor. The repo contains the content for [docs.d-sports.org](https://docs.d-sports.org).
  </Accordion>
  <Accordion icon="rectangle-terminal" title="Start the docs preview">
    1. Install the Mintlify CLI: `npm i -g mint`
    2. In the docs directory (where `docs.json` lives), run: `mint dev`
    3. Open http://localhost:3000 to see the docs.

    <Tip>Changes to MDX and `docs.json` update automatically in the preview.</Tip>
  </Accordion>
</AccordionGroup>

### Step 2: (Optional) Run an app

To run one of the D-Sports applications (PWA, marketing site, native app, or Mic'd Up), use the repo-specific instructions:

- **[d-sports-api](/repositories/d-sports-api)** — `bun dev` (port 3004)
- **[d-sports-site](/repositories/d-sports-site)** — `bun dev` (port 3005)
- **[d-sports-engage-native](/repositories/d-sports-engage-native)** — `bunx expo start`
- **[d-sports-mic-d-up](/repositories/d-sports-mic-d-up)** — backend in `backend/`, then mobile with `bun run android` or `bun run ios`

See the [Ecosystem overview](/repositories/ecosystem-overview) and each repo page for prerequisites (Node, Bun, PostgreSQL, env vars).

### Step 3: Deploy docs changes

1. Commit and push your changes to the default branch.
2. If the D-Sports docs are connected via the Mintlify GitHub app, the site at [docs.d-sports.org](https://docs.d-sports.org) updates automatically after the push.
